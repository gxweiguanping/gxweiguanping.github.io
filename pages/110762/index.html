# IDEA常用插件

插件名称 作用
CodeGlance 拖动浏览代码更加方便，还有放大镜功能。
Rainbow Brackets 它可以实现配对括号相同颜色，并且实现选中区域代码高亮的功能。对增强写代码的有趣性和排错等都有一些帮助。
Translation 很不错的翻译插件，支持谷歌、有道、百度三种翻译。
Alibaba Java Code Guidelines 阿里巴巴出品的代码规范插件，用于帮助程序员规范自己的代码，检测出潜在的问题，改善代码质量。
Maven Helper 查看maven的依赖树。
MybatisX MybatisX 是一款基于 IDEA 的快速开发插件，为效率而生。最强拍档：搭配Mybatis-Plus开发。具备XML 跳转功能。
git commit template git提交规范插件。
Mybatis Log Free 用于记录MyBatis执行的SQL语句。
PlantUML Integration PlantUml画图利器。
Smart Input 注释的时候能自动切换中文，编写代码的时候自动切换英文的插件。
Grep Console 给控制台日志不同的颜色，日志管理神器。

# 参数详解

type(必须)：用于说明git commit的类别

可选参数 说明
Features 新功能（feature）
Bug Fixes 修复bug，可以是QA发现的BUG，也可以是研发自己发现的BUG
Documentation 更新了文档，或者更新了注释
Styles 格式（不影响代码运行的变动）比如执行了format、更改了tab显示等
Code Refactoring 重构（即不是新增功能，也不是修改bug的代码变动）指的是代码结构的调整，比如使用了一些设计模式重新组织了代码
Performance lmprovements 优化相关，比如提升性能、体验
Tests 增加了单元测试和自动化相关的代码
Builds 影响编译的一些更改，比如更改了maven插件、增加了npm的过程等
Continuous Integrations 持续集成方面的更改。现在有些build系统喜欢把ci功能使用yml描述。如有这种更改，建议使用ci
Chores 其他改动。比如一些注释修改或者文件清理。不影响src和test代码文件的，都可以放在这里
Reverts 回滚到上一个版本

scope(必须) scope用于说明 commit 影响的范围，比如数据层、控制层、视图层等等，视项目不同而不同。

Short descriptiont(必须) Short description是commit的简短描述，不超过50个字符。
