# maven私服

私服（Private Repository）是部署在局域网内的特殊远程仓库，用于代理和管理项目构建所需的依赖库和第三方构件。其主要目的是优化依赖管理流程，提高构建和部署效率。

通过私服，可以实现以下功能：

  1. 依赖代理和缓存： 私服作为一个中间层，可以代理访问外部的远程仓库，缓存下载过的构件。当项目构建需要依赖库时，首先在私服中查找，如果有则直接下载，避免频繁访问外部仓库，加快构建速度。
  2. 提供内部构件： 私服可以存储和管理内部开发的构件，团队成员可以将自己开发的库上传到私服，供团队内部共享使用。
  3. 第三方构件管理： 私服也可以用于存储和管理第三方的构件，这些构件可能无法直接从公共仓库获取，或者为了确保稳定性和安全性而选择存储在私服中。
  4. 镜像和高可用： 私服可以作为远程仓库的镜像，提供高可用性和稳定的访问，减少外部仓库不稳定性对项目构建的影响。

Nexus 是一个流行的 Maven 仓库管理器，能够简化私服的搭建和维护。它提供了图形化界面来管理仓库、构件、权限等，支持缓存、镜像和代理等功能，大大提高了团队的开发效率和构建稳定性。

# 仓库概念

仓库名称 描述 默认地址 策略
maven-central Maven中央库，代理Maven中央仓库 https://repo1.maven.org/maven2/ (opens new window) Release
maven-releases 私有仓库，用于存储发行版jar包 配置自定义地址 Release
maven-snapshots 私有仓库，用于存储快照（调试版本）jar包 配置自定义地址 Snapshot
maven-public 仓库分组，包含上述三个仓库，对外提供服务 配置自定义地址，或组合maven-central和maven-releases等地址 -

在Maven构建中，这些仓库的作用如下：

  1. maven-central： 代理Maven中央仓库，用于下载已发布的稳定版本的构件。默认从 https://repo1.maven.org/maven2/ (opens new window)拉取jar包，适用于生产环境。
  2. maven-releases： 私有仓库，用于存储发行版（Release）的jar包，包含项目的正式版本。可以配置自定义地址，方便团队内部管理和共享。
  3. maven-snapshots： 私有仓库，用于存储快照（Snapshot）版本的jar包，这些版本通常用于开发和调试。同样可以配置自定义地址。
  4. maven-public： 仓库分组，将maven-central、maven-releases和maven-snapshots这三个仓库组合在一起，对外提供服务。可以将组合后的仓库配置在Maven的 settings.xml 文件中，以便在项目中统一管理仓库地址。

# Nexus默认的仓库类型

仓库类型 描述
group（仓库组类型） 用于创建开发人员自定义的仓库组，将多个仓库组合在一起。可以将多个仓库的访问组合成一个逻辑上的仓库。
hosted（宿主类型） 用于存放内部项目的发布版（Release）jar包，开发人员发布的稳定版本将被存储在这个仓库中。
proxy（代理类型） 用于从远程仓库中寻找和缓存数据，避免重复下载，提高构建效率。通常用于代理远程中央仓库或其他公共仓库。
virtual（虚拟类型） 用于将多个仓库（宿主或代理）组合成一个虚拟仓库，使开发人员只需关注一个虚拟仓库，而无需关心底层实际仓库。

Nexus 默认的仓库类型包括了仓库组类型（group）、宿主类型（hosted）、代理类型（proxy）以及虚拟类型（virtual）。这些类型的仓库可以根据项目需要灵活配置和组合，以满足不同场景下的依赖管理和构建需求。

# Nexus仓库分类的概念

仓库分类概念 描述
直接访问宿主仓库 Maven 可直接从宿主仓库中下载构件，宿主仓库存放着内部发布的稳定版本构件。
代理仓库下载缓存构件 代理仓库间接地从远程仓库下载构件，并将下载的构件缓存起来，提高构建效率。
仓库组访问 为了方便 Maven 下载构件，可以创建仓库组，仓库组本身没有实际内容，访问时会转向包含的宿主或代理仓库获得实际构件内容。

# 上传本地仓库的jar包到私服

1、修改本地maven配置文件settings.xml

<servers>
	<server>
      <id>releases</id>
      <username>admin</username>
      <password>admin123456</password>
    </server>
    <server>
      <id>snapshots</id>
      <username>admin</username>
      <password>admin123456</password>
    </server>
 </servers>

# 从私服下载jar包

修改本地maven配置文件settings.xml

<mirror>
      <id>test</id>
      <mirrorOf>central</mirrorOf>  # 固定写死
      <name>Nexus</name>
      <url>http://192.168.1.29:8866/repository/maven-public/</url>
</mirror>

提示

有时候快照仓库上的包拉不下来，需要在pom.xml中加上以下配置。

<repositories>
    <repository>
        <id>nexus-maven</id>
        <name>nexus maven</name>
        <url>http://192.168.1.29:8866/repository/maven-public/</url>
        <releases>
            <enabled>true</enabled>
            <updatePolicy>always</updatePolicy>
        </releases>
        <snapshots>
            <enabled>true</enabled>
            <updatePolicy>always</updatePolicy>
        </snapshots>
    </repository>
</repositories>

具体解释如下：

  • <repository>：定义一个仓库配置。
  • <id>：仓库的唯一标识符，用于在其他地方引用这个仓库。
  • <name>：仓库的名称，可以是任意描述性文本。
  • <url>：仓库的地址，即从哪个 URL 下载依赖构件。
  • <releases>：用于配置仓库的发布版本策略。
    • <enabled>：是否启用发布版本下载，这里设置为 true 表示启用。
    • <updatePolicy>：更新策略，always 表示始终尝试下载最新版本的发布构件。
  • <snapshots>：用于配置仓库的快照版本策略。
    • <enabled>：是否启用快照版本下载，这里设置为 true 表示启用。
    • <updatePolicy>：更新策略，always 表示始终尝试下载最新版本的快照构件。

这段配置指定了一个名为 nexus-maven的仓库，地址为 http://192.168.1.29:8866/repository/maven-public/ (opens new window)，并启用了发布版本和快照版本的下载，并且都设置为始终尝试下载最新版本。

通过在 Maven 项目的 pom.xml 文件中添加这样的 <repositories> 配置，就可以让 Maven 在构建过程中从指定的 Nexus 仓库获取所需的依赖构件。
