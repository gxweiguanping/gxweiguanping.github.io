离线上传镜像包到docker

# 离线上传镜像包到docker

# 步骤1：先在网络的的docker上面pull需要的镜像

docker pull mysql:8.0
1

# 步骤2：导出镜像

docker save -o mysql8.tar mysql:8.0

# mysql8.tar tar的名称
# mysql:8.0  mysql是镜像的名称，8.0是版本
1
2
3
4
  • docker save: 这是 Docker 命令行接口的一个命令，用于保存一个或多个 Docker 镜像到一个 tar 文件。
  • -o mysql8.tar: -o 是 --output 的缩写，这个参数指定了输出文件的路径和名字。在这个例子中，输出文件的名字为 mysql8.tar。
  • mysql:8.0: 这是你想要保存的 Docker 镜像的名字和标签。在这个例子中，我们保存的是 MySQL 8.0 的镜像。

运行这个命令后，你会在当前目录下得到一个名为 mysql8.tar 的文件，这个文件包含了 MySQL 8.0 Docker 镜像的所有层和元数据。

你可以将这个 tar 文件复制到其他机器上，然后使用 docker load 命令来导入 Docker 镜像。

# 步骤3：将镜像包mysql8.tar上传至无网络的环境上，执行

docker load -i mysql8.tar
1
  • docker load: 这是 Docker 命令行接口的一个命令，用于从一个 tar 文件或者标准输入（STDIN）中导入一个或多个 Docker 镜像。
  • -i mysql8.tar: -i 是 --input 的缩写，这个参数指定了输入文件的路径和名字。在这个例子中，输入文件的名字为 mysql8.tar。

这个命令会从 mysql8.tar 文件中导入 Docker 镜像到本地 Docker 环境。在这个例子中，导入的是之前通过 docker save 命令保存的 MySQL 8.0 Docker 镜像。

导入完成后，你可以使用 docker images 命令查看本地 Docker 环境中的所有 Docker 镜像，你应该能看到导入的 MySQL 8.0 Docker 镜像在列表中。
