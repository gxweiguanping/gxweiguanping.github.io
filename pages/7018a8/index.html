# SQL LIKE 操作符

SQL 的 LIKE 操作符用于在 WHERE 子句中搜索列中的指定模式。语法为：

SELECT column1, column2, ...
FROM table_name
WHERE columnN LIKE pattern;
1
2
3

这里的 "pattern" 可以包含两种通配符：

  1. %：表示任意数量（包括 0）的任意字符。例如，'a%' 匹配以 'a' 开始的任何字符串，'%a' 匹配以 'a' 结束的任何字符串，'%a%' 匹配包含 'a' 的任何字符串。
  2. _：表示任意单个字符。例如，'a_' 匹配以 'a' 开始的任何两个字符的字符串，'_a' 匹配以 'a' 结束的任何两个字符的字符串，'_a_' 匹配包含 'a' 的任何三个字符的字符串。

这是一个使用 LIKE 的示例：

SELECT * FROM Employees
WHERE FirstName LIKE 'J%';
1
2

这将选择 "Employees" 表中 FirstName 开头为 'J' 的所有行。

注意，LIKE 操作符是大小写敏感的，这意味着 'a%' 和 'A%' 会返回不同的结果。如果你希望进行大小写不敏感的匹配，可以使用函数 LOWER() 或 UPPER() 将列值和模式转换为相同的大小写。

LIKE CONCAT 和 LIKE 是在 SQL 中用于模糊匹配的操作符，它们之间的区别如下：

  1. LIKE：LIKE 是用于模糊匹配的关键字，它可以在 WHERE 子句中与通配符配合使用，来匹配符合指定模式的字符串。例如，column LIKE 'abc%' 表示匹配以 "abc" 开头的字符串。
  2. CONCAT：CONCAT 是 SQL 中的一个函数，用于连接多个字符串。它将多个字符串拼接在一起，生成一个新的字符串。例如，CONCAT('abc', 'def') 返回结果为 "abcdef"。

当涉及到模糊匹配时，有时候我们需要动态构造模糊匹配的模式。这时可以使用 LIKE CONCAT 来实现动态拼接模糊匹配的字符串。通过将 LIKE 和 CONCAT 结合使用，可以将模糊匹配的模式动态生成。

例如，如果要匹配以特定字符串开头并以特定字符串结尾的情况，可以使用 LIKE CONCAT，如 column LIKE CONCAT('abc', '%')。这将匹配以 "abc" 开头的字符串。

总结起来，LIKE 是用于模糊匹配的操作符，而 CONCAT 是用于字符串拼接的函数。LIKE CONCAT 可以动态构造模糊匹配的模式。

在SQL中，索引用于快速查找和检索数据库表中的数据。当你在一列上创建索引时，数据库会创建一个快速查找表来帮助它更快地查找数据。

当使用 LIKE 语句时，是否可以使用索引取决于你的模式（pattern）如何开始：

  1. 当你的模式以通配符开始，例如 %abc，SQL 无法使用索引。这是因为索引通常是基于列值的开始部分构建的，如果模式以通配符开始，那么数据库无法使用索引来提高查询性能。
  2. 相反，如果你的模式不是以通配符开始，例如 abc% 或 abc%def，SQL 可以利用索引来加速查询。这是因为在这种情况下，数据库可以使用索引来快速定位满足模式开始部分的行，然后检查这些行是否满足整个模式。

总的来说，如果你想在使用 LIKE 查询时获得索引的优势，那么你的模式不应该以通配符开始。这是一个通常在数据库优化中使用的策略。然而，实际情况可能会更复杂，因为是否使用索引还取决于其他因素，例如数据库管理系统（DBMS）的实现、表的大小、索引的类型（例如，B-tree、hash、bitmap 等）、索引的选择性以及查询的其他部分。对于复杂的查询和大型数据库，通常需要数据库管理员或者有经验的开发者来确定最优的索引策略。

# SQL IN & NOT IN操作符

IN 是 SQL 中的一个操作符，用于在 WHERE 子句中指定多个可能的值。你可以用它来测试列中的值是否在一组指定的值中。

下面是一些使用 IN 操作符的例子：

  1. 基本用法：
SELECT * FROM Customers
WHERE Country IN ('Germany', 'France', 'UK');
1
2

上述查询从 Customers 表中选取所有国家（Country）是 "Germany"，"France" 或 "UK" 的记录。

  1. 与子查询一起使用：
SELECT * FROM Customers
WHERE CustomerID IN (SELECT CustomerID FROM Orders WHERE Quantity > 10);
1
2

这个查询从 Customers 表中选取所有的客户，这些客户在 Orders 表中有至少一条订单的数量（Quantity）大于10。

  1. NOT IN 用法：

你也可以使用 NOT IN 来选取不在指定值集合中的记录。

SELECT * FROM Customers
WHERE Country NOT IN ('Germany', 'France', 'UK');
1
2

上述查询从 Customers 表中选取所有国家（Country）不是 "Germany"，"France" 或 "UK" 的记录。

使用 IN 操作符可以使你的 SQL 语句更清晰、更简洁，特别是当你需要匹配多个值时。

# SQL EXISTS

EXISTS 是 SQL 中用于判断子查询是否返回结果的关键字。它通常与 SELECT 语句结合使用，用于检查主查询中的条件是否满足子查询的结果。

EXISTS 关键字的语法如下：

EXISTS (subquery)
1

其中，subquery 是一个子查询，它可以是一个完整的 SELECT 语句或其他返回结果集的子查询。

EXISTS 的工作原理如下：

  1. 首先，执行子查询，并获取其结果集。
  2. 如果子查询返回至少一行结果，则 EXISTS 返回 TRUE，表示主查询中的条件满足。
  3. 如果子查询没有返回任何结果，则 EXISTS 返回 FALSE，表示主查询中的条件不满足。

在使用 EXISTS 时，常见的用法是将子查询的结果与主查询中的条件进行关联，以实现更复杂的查询逻辑。通常，子查询中的列将与主查询中的列进行比较，以确定是否存在满足条件的记录。

以下是一个示例，演示了如何使用 EXISTS：

sqlCopy codeSELECT column1, column2, ...
FROM table1
WHERE EXISTS (
    SELECT column3
    FROM table2
    WHERE table1.column1 = table2.column3
)

在上述示例中，主查询从 table1 表中选择列 column1、column2 等，并通过 EXISTS 判断是否存在满足条件的记录。子查询从 table2 表中选择列 column3，并将其与主查询中的条件进行比较。

EXISTS 的优点是可以实现高效的查询，因为它只需要判断子查询是否返回结果，而不需要获取所有的子查询结果。这在处理大型数据集时尤为重要。

需要注意的是，EXISTS 只返回布尔值（TRUE 或 FALSE），而不返回实际的结果集。它通常用于条件判断、子查询优化和相关查询等场景。

# SQL BETWEEN & NOT BETWEEN 操作符

BETWEEN 是 SQL 中的一个操作符，它用于在 WHERE 子句中选择给定范围内的值。范围的边界包含在内。

这是一个简单的 BETWEEN 语句的例子：

SELECT * FROM Products
WHERE Price BETWEEN 10 AND 20;
1
2

这个查询从 Products 表中选择价格 Price 在10到20之间的所有产品。

你也可以使用 BETWEEN 与日期一起使用：

SELECT * FROM Orders
WHERE OrderDate BETWEEN '2023-01-01' AND '2023-12-31';
1
2

这个查询从 Orders 表中选取 OrderDate 在2023年之间的所有订单。

如果你需要选择在某个范围之外的值，你可以使用 NOT BETWEEN：

SELECT * FROM Products
WHERE Price NOT BETWEEN 10 AND 20;
1
2

这个查询从 Products 表中选择价格 Price 不在10到20之间的所有产品。

需要注意的是，BETWEEN 操作符是闭区间，包含区间的边界。即，上述例子中的价格等于10或者20的产品也会被选取。

# SQL 别名

在 SQL 中，"别名"是一个由用户定义的替代名，用于给表或者列起一个新名字。别名可以在查询中提高可读性，或者在连接多个表时消除任何可能的歧义。

列别名： 你可以给查询结果的列赋一个别名，这会在结果中改变列的头部显示。比如：

SELECT CustomerID as ID, CustomerName as Customer
FROM Customers;
1
2

在这个查询中，CustomerID 列将显示为 ID，CustomerName 列将显示为 Customer。

表别名： 在更复杂的查询中，特别是涉及到多个表的联接时，表别名非常有用。比如：

SELECT o.OrderID, o.OrderDate, c.CustomerName
FROM Customers AS c
JOIN Orders AS o ON c.CustomerID = o.CustomerID;
1
2
3

在这个查询中，Customers 表被别名为 c，Orders 表被别名为 o。这样在查询的其他部分，就可以使用这些更简洁的别名，而不是原始的表名。

# SQL INNER JOIN 关键字

在 SQL 中，JOIN 语句用于在两个或更多的表之间组合行，基于这些表之间的相关列之间的关系。

有多种类型的 SQL 连接，这里是最常见的几种：

INNER JOIN：INNER JOIN 关键字选择那些在两个或多个表中都有匹配的记录的记录。如果表中的任一行在另一表中没有匹配，那么结果是没有这些记录的行。例如：

SELECT Orders.OrderID, Customers.CustomerName
FROM Orders
INNER JOIN Customers ON Orders.CustomerID = Customers.CustomerID;
1
2
3

在这个例子中，INNER JOIN 返回了 Orders 表中的 CustomerID 与 Customers 表中的 CustomerID 匹配的所有记录。

# SQL LEFT JOIN 关键字

LEFT JOIN：LEFT JOIN 关键字从第一个（左边）表返回所有的行，即使右边的表中没有匹配的行。例如：

SELECT Customers.CustomerName, Orders.OrderID
FROM Customers
LEFT JOIN Orders ON Customers.CustomerID = Orders.CustomerID;
1
2
3

这个查询将返回 Customers 表的所有记录，即使 Orders 表中没有匹配的记录。

# SQL RIGHT JOIN 关键字

RIGHT JOIN：RIGHT JOIN 关键字从第二个（右边）表返回所有的行，即使左边的表中没有匹配的行。

# SQL NOT NULL 约束

在 SQL 中，NOT NULL 约束确保列不能有 NULL 值。

在创建表时，你可以使用 NOT NULL 约束。例如：

CREATE TABLE Customers (
    ID int NOT NULL,
    Name varchar(255) NOT NULL,
    Email varchar(255),
    Address varchar(255),
    City varchar(255)
);
1
2
3
4
5
6

在这个例子中，ID 和 Name 列不能接受 NULL 值，但其他列可以。

如果你试图在 NOT NULL 约束的列中插入 NULL 值，将会收到错误。

你也可以使用 ALTER TABLE 语句在已有的表上添加 NOT NULL 约束。例如：

ALTER TABLE Customers
MODIFY Name varchar(255) NOT NULL;
1
2

在这个例子中，现有的 Customers 表的 Name 列被修改为不接受 NULL 值。如果 Name 列中已有 NULL 值，你将无法添加 NOT NULL 约束，除非你首先更改这些 NULL 值。

# SQL UNIQUE 约束SQL

在 SQL 中，UNIQUE 约束确保所有在一列中的值都是唯一的。

当创建表时，你可以使用 UNIQUE 约束，例如：

CREATE TABLE Customers (
    ID int NOT NULL,
    Email varchar(255) UNIQUE,
    Name varchar(255),
    Address varchar(255),
    City varchar(255)
);

2
3
4

在这个例子中，Email 列必须包含唯一值。如果你尝试插入与现有值重复的值，你会收到一个错误。

你也可以在已存在的表上添加 UNIQUE 约束。例如：

ALTER TABLE Customers
ADD UNIQUE (Email);
1
2

在这个例子中，Customers 表的 Email 列被添加了 UNIQUE 约束。如果 Email 列中已有重复值，你将无法添加 UNIQUE 约束，除非你首先更改这些重复的值。

还可以在多个列上定义一个 UNIQUE 约束，这被称为复合 UNIQUE 约束。例如：

CREATE TABLE Customers (
    ID int NOT NULL,
    FirstName varchar(255),
    LastName varchar(255),
    CONSTRAINT uc_Customer UNIQUE (FirstName,LastName)
);

在这个例子中，FirstName 和 LastName 列的组合必须是唯一的。

# CREATE INDEX 语句

在 SQL 中，索引用于加速查询性能，特别是对于大量数据的表，索引可以极大地提高查询效率。

CREATE INDEX 语句用于在一个表上创建索引。基本的语法如下：

CREATE INDEX index_name
ON table_name (column1, column2, ...);
1
2
  • index_name 是你为新索引选择的名字。
  • table_name 是你想在上面创建索引的表的名字。
  • column1, column2, ... 是你想在上面创建索引的列的名字。

例如：

CREATE INDEX idx_lastname
ON Customers (LastName);
1
2

在这个例子中，我们在 Customers 表的 LastName 列上创建了一个名为 idx_lastname 的索引。这意味着数据库会对 LastName 列进行排序，并为每个值保留一个指针，从而加快对 LastName 列的查询。

注意，索引可以极大地提高查询速度，但创建和维护索引也需要时间，特别是对于大量数据的表，插入、更新和删除操作可能会变慢，因为每次对表的修改都需要更新索引。因此，是否创建索引需要根据具体情况权衡。

此外，还有一些特殊类型的索引，例如唯一索引、全文索引等，它们的创建方式略有不同，你可能需要查阅具体的数据库文档来了解如何创建这些特殊类型的索引。

# WITH RECURSIVE递归

WITH RECURSIVE 是一种在 SQL 查询中使用的关键字组合，用于创建递归公共表达式（Recursive CTE，Common Table Expression）。递归公共表达式允许您在查询中定义一个递归结构，通过多次迭代自身来构建结果集，特别适用于处理层次性数据结构，例如树形结构或图形结构。

基本语法如下：

WITH RECURSIVE cte_name (column_list) AS (
    -- 初始查询部分
    SELECT ...
    UNION [ALL]
    -- 递归查询部分
    SELECT ...
    FROM cte_name JOIN ...
)
SELECT * FROM cte_name;

  • column_list：指定要返回的列。
  • 初始查询部分：在 SELECT ... 子句中定义初始查询。
  • 递归查询部分：在 SELECT ... FROM cte_name JOIN ... 子句中定义递归查询，通常是将 cte_name 与其他表联接，并定义终止条件。
