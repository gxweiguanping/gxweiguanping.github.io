基本Git命令

# 基本Git命令

# 初始化仓库

在项目文件夹中运行以下命令来创建一个新的Git仓库：

git init
1

# 克隆仓库

从远程仓库克隆一个项目：

git clone <repository-url>
1

git clone <repository-url> 命令用于从远程 Git 仓库克隆一个副本到本地计算机。参数的含义如下：

  • <repository-url>: 这是远程 Git 仓库的 URL。它可以是一个 HTTP(S) 或 SSH 地址，指向您要克隆的远程仓库。例如，https://github.com/user/repo.git 或 git@github.com:user/repo.git。

例如，如果您要克隆一个 URL 为 https://github.com/user/repo.git 的远程仓库，您可以使用以下命令：

git clone https://github.com/user/repo.git
1

执行此命令后，Git 将在当前目录下创建一个名为 repo 的新目录，并将远程仓库的所有文件、分支和提交历史复制到该目录中。接下来，您可以进入 repo 目录，在本地进行修改、提交和与远程仓库的同步。

# 添加和提交文件

添加文件到暂存区：

git add <file-name>
1

git add <file-name> 命令用于将指定文件添加到 Git 的暂存区（staging area），以便在下次提交时包含这些更改。参数的含义如下：

  • <file-name>：这是要添加到暂存区的文件名。您可以使用相对或绝对路径指定文件。也可以使用通配符 * 添加多个文件或目录。

例如，如果您要将名为 example.txt 的文件添加到暂存区，可以使用以下命令：

git add example.txt
1

如果您要一次性将多个文件添加到暂存区，可以使用逗号分隔的文件列表或通配符 *。例如，将所有 .txt 文件添加到暂存区：

git add *.txt
1

在执行 git commit 命令时，暂存区中的所有更改将被记录为一个新的提交。通过使用 git add 命令，您可以选择性地添加一部分更改而不是所有更改，以便创建更小、更具描述性的提交。

添加所有文件到暂存区：

git add .
1

提交更改：

git commit -m "Commit message"
1

# 查看状态和历史

查看仓库状态：

git status
1

查看提交历史：

git log
1

# 分支管理

创建新分支：

git branch <branch-name>
1

git branch <branch-name> 命令用于在当前 Git 仓库中创建一个新的分支。参数的含义如下：

  • <branch-name>：这是您要创建的新分支的名称。

例如，如果您想要创建一个名为 feature-x 的新分支，您可以使用以下命令：

git branch feature-x
1

这将在当前的提交（HEAD）下创建一个新的分支。创建新分支后，您可以使用 git checkout 命令切换到新分支进行开发。

需要注意的是，git branch 命令只是创建一个新分支，并不会自动切换到新分支。要同时创建并切换到新分支，您可以使用 git checkout -b <branch-name> 命令。

切换到其他分支：

git checkout <branch-name>
1

git checkout <branch-name> 命令用于在 Git 仓库中切换到指定的分支。参数的含义如下：

  • <branch-name>：这是您要切换到的分支的名称。

例如，如果您想要切换到名为 feature-x 的分支，您可以使用以下命令：

git checkout feature-x
1

执行此命令后，您将切换到 feature-x 分支，并且您的工作目录中的文件将更新为该分支的最新提交。在这个新分支上进行的任何修改和提交都将仅在该分支上进行，不会影响其他分支。

需要注意的是，如果您在切换分支前对工作目录中的文件进行了修改，但尚未提交，Git 会提醒您提交更改，或者使用 git stash 将更改暂存起来，以便在切换分支后还能恢复这些更改。

创建并切换到新分支：

git checkout -b <branch-name>
1

git checkout -b <branch-name> 命令用于在 Git 仓库中创建一个新的分支，并立即切换到这个新分支。参数的含义如下：

  1. <branch-name>：这是您要创建并切换到的新分支的名称。

例如，如果您想要创建一个名为 feature-x 的新分支，并立即切换到这个新分支，您可以使用以下命令：

git checkout -b feature-x
1

执行此命令后，Git 将基于当前分支创建一个新的 feature-x 分支，并将工作目录中的文件更新为该分支的最新提交。在这个新分支上进行的任何修改和提交都将仅在该分支上进行，不会影响其他分支。

这个命令等同于先执行 git branch <branch-name> 创建新分支，然后执行 git checkout <branch-name> 切换到新分支。

列出所有分支：

git branch
1

合并分支：

git merge <branch-name>
1

git merge <branch-name> 命令用于将指定分支的更改合并到当前分支。参数的含义如下：

  • <branch-name>：这是您要合并到当前分支的源分支的名称。

例如，如果您当前位于 master 分支，并希望合并 feature-x 分支的更改，您可以使用以下命令：

git merge feature-x
1

执行此命令后，Git 将尝试自动合并两个分支的更改。如果在合并过程中遇到冲突（即，两个分支对同一文件的同一部分进行了不同的更改），则需要手动解决冲突并提交更改以完成合并操作。

git merge 默认使用"快进合并"（fast-forward merge）策略，如果可能的话。这意味着，如果当前分支的提交是要合并分支的提交的直接祖先，Git 会简单地将当前分支指针移动到要合并分支的最新提交。否则，Git 会创建一个新的合并提交，它具有两个父提交（即，当前分支和要合并分支的最新提交）。

删除分支：

git branch -d <branch-name>
1

git branch -d <branch-name> 命令用于删除一个本地分支。其中 <branch-name> 是你想要删除的分支的名称。请注意，你不能删除当前已检出的分支。在删除分支之前，确保已经将所需的更改合并到其他分支，因为删除分支将丢失该分支上的所有更改。

如果要强制删除分支，即使它尚未合并到其他分支，可以使用：

git branch -D <branch-name>
1

这将无条件地删除分支，无论其合并状态如何。请谨慎使用此选项。

# 远程仓库

添加远程仓库：

git remote add <remote-name> <repository-url>
1
  • <remote-name>: 这是您要添加的远程仓库的名称。为了方便使用，您可以为远程仓库选择一个简短且易于记忆的名称。例如，您可以使用 upstream 作为名称，表示上游仓库。
  • <repository-url>: 这是远程 Git 仓库的 URL。它可以是一个 HTTP(S) 或 SSH 地址，指向您要添加的远程仓库。例如，https://github.com/user/repo.git 或 git@github.com:user/repo.git。

例如，如果您要添加一个名为 upstream 的远程仓库，其 URL 为 https://github.com/user/repo.git，您可以使用以下命令：

git remote add upstream https://github.com/user/repo.git
1

执行此命令后，您可以使用 upstream 名称引用该远程仓库，并使用其他 Git 命令（如 git fetch、git pull、git push 等）与该远程仓库进行交互。

查看远程仓库列表：

git remote -v
1

拉取远程仓库的更改：

git pull <remote-name> <branch-name>
1
  • <remote-name>: 这是您要拉取更改的远程仓库的名称。在克隆一个 Git 仓库时，默认的远程仓库名称为 origin。您可以通过运行 git remote 命令查看当前 Git 仓库中已配置的所有远程仓库。
  • <branch-name>: 这是您要从远程仓库拉取更改的分支名称。例如，如果您想要获取远程仓库中 main 分支的更改，那么 <branch-name> 就是 main。

举个例子，如果您想要从名为 origin 的远程仓库拉取 main 分支的更改，并将其合并到您当前所在的分支，您可以使用以下命令：

git pull origin main
1

这个命令将获取远程仓库 origin 中 main 分支上的更改，并将其合并到您当前所在的分支。这样，您可以在本地分支上查看并处理远程分支的更新。

推送本地更改到远程仓库：

git push <remote-name> <branch-name>
1
  • <remote-name>: 这是您要推送到的远程仓库的名称。在克隆一个 Git 仓库时，默认的远程仓库名称为 origin。您可以通过运行 git remote 命令查看当前 Git 仓库中已配置的所有远程仓库。
  • <branch-name>: 这是您要推送的本地分支名称。例如，如果您当前所在的分支是 feature，并且您希望将更改推送到远程仓库，那么 <branch-name> 就是 feature。

举个例子，如果您想要将本地的 my-feature-branch 分支推送到名为 origin 的远程仓库，您可以使用以下命令：

git push origin my-feature-branch
1

这个命令将把您在 my-feature-branch 分支上所做的更改推送到 origin 远程仓库。远程仓库中的其他人将能够看到您的更改并与您

的分支进行协作。

删除远程仓库：

git remote rm <remote-name>
1
  • <remote-name> 参数是您要删除的远程仓库的名称。

例如，当您想要删除一个名为 upstream 的远程仓库时，可以使用以下命令：

git remote rm upstream
1

# 撤销更改

撤销工作区的更改：

git checkout -- <file-name>
1

移除暂存区的更改：

git reset HEAD <file-name>
1

撤销上一次提交（创建一个新的提交以撤销之前的提交）：

git revert <commit-hash>
1

重置到特定提交（谨慎使用，会丢失所有后续提交）：

git reset --hard <commit-hash>
1

# 回滚到之前某一版本且该版本后的提交都不需要

1）首先使用git log命令查看版本号(commit后就是版本号)

commit ec32027ecbaa90e4c89662440117a6b9c9293c3c
1

2）git reset --hard 版本号 回滚到之前某版本，此时本地已经回滚，远程仓库还未回滚

  • git reset 是用于移动 HEAD 指针的命令，HEAD 是指向当前所在分支的指针。
  • --hard 选项表示重置后，将工作目录、暂存区和本地仓库全部恢复到指定版本，即丢弃之前的所有更改。

3）git push -f 回滚远程仓库版本

  • git push 是用于将本地更改推送到远程仓库的命令。
  • -f 或 --force 选项表示强制推送，它用于覆盖远程仓库上的历史记录。

# git commit但未push 把修改撤销到工作区中

使用 git reset --soft HEAD^ 命令，撤销了commit 代码依然保留

git reset --soft HEAD^ 是用于取消最后一次提交并将提交的更改保留在工作区中的 Git 命令。它将 HEAD 指针（当前分支的最新提交）向后移动一个提交，取消最后一次提交，但保留提交的更改在工作区中。

  • git reset 用于移动分支的指针。
  • --soft 选项表示重置将保留提交的更改在工作区中。
  • HEAD^ 表示将 HEAD 指针向后移动一个提交，即取消最后一次提交。

执行此命令后，你的更改将在工作区中保留，你可以对它们进行修改并重新提交，而不需要重新创建这些更改。这在需要修复或修改最后一次提交时非常有用。

请注意，如果你希望取消并删除最后一次提交，可以使用 --hard 选项而不是 --soft
