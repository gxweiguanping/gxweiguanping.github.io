# MySQL基础语法教程

# SQL SELECT

SQL SELECT语句用于从数据库表中选择并返回数据。其基本格式如下：

SELECT column1, column2, ...
FROM table_name;
1
2

这里，你可以在SELECT后面列出你想要选择的列。如果你想选择所有列，可以使用*代替列名，如下所示：

SELECT * 
FROM table_name;
1
2

也可以在SELECT语句中使用许多其他子句，以进一步改变你选择的数据，包括：

  • WHERE：基于某些条件筛选行。例如，SELECT * FROM Students WHERE Age > 18;只会选择年龄大于18的学生。
  • GROUP BY：基于一列或多列的值对结果进行分组。例如，SELECT COUNT(*), Country FROM Students GROUP BY Country;将计算每个国家的学生数量。
  • HAVING：与GROUP BY结合使用，基于组的属性筛选组。例如，SELECT COUNT(*), Country FROM Students GROUP BY Country HAVING COUNT(*) > 100;将只选择有超过100名学生的国家。
  • ORDER BY：按照一列或多列的值对结果进行排序。例如，SELECT * FROM Students ORDER BY Age DESC;将按照年龄的降序返回学生。
  • JOIN：可以将两个或更多的表连接在一起，基于某些列的值匹配行。例如，SELECT Students.Name, Courses.Name FROM Students JOIN Courses ON Students.CourseID = Courses.CourseID;将连接学生和课程表，显示每个学生所上的课程。

这些是最基本的SQL SELECT语句和子句，但是实际上，SQL语言非常强大和灵活，可以执行非常复杂的查询操作。

# SQL SELECT DISTINCT

在SQL中，SELECT DISTINCT 语句用于返回唯一的（不同的）值。

在表中，一列可能会包含许多重复值。有时，您可能只希望列出不同（distinct）的值。

SELECT DISTINCT语句的语法如下：

SELECT DISTINCT column1, column2, ...
FROM table_name;
1
2

这里是一个例子。假设我们有一个名为 Orders 的表，其中包含 Customer 列，其中包含很多重复的值。如果我们只想找出所有不同的客户，我们可以使用 SELECT DISTINCT，如下所示：

SELECT DISTINCT Customer
FROM Orders;
1
2

这将返回 Customer 列中的所有不同值。

注意，SELECT DISTINCT 在所有选定的列上执行。这意味着，如果你选择了多列，那么这个组合的每个不同组合都将作为一个唯一的行返回。例如：

SELECT DISTINCT column1, column2
FROM table_name;
1
2

这将返回 column1 和 column2 的所有独特组合。

# SQL WHERE

在 SQL 中，WHERE 子句用于过滤记录，返回满足特定条件的记录。

WHERE 子句的语法如下：

SELECT column1, column2, ...
FROM table_name
WHERE condition;
1
2
3

这里，condition 是你要记录满足的条件。你可以使用各种比较操作符，如 =（等于）、<> 或 !=（不等于）、>（大于）、<（小于）、>=（大于或等于）、<=（小于或等于）。

例如，假设我们有一个名为 "Employees" 的表，我们只想选择年龄大于30的员工：

SELECT * 
FROM Employees 
WHERE Age > 30;
1
2
3

你还可以使用 AND、OR 和 NOT 运算符组合多个条件。

例如，如果你想选择年龄在 30 到 40 之间的员工，你可以这样写：

SELECT * 
FROM Employees 
WHERE Age >= 30 AND Age <= 40;
1
2
3

如果你想选择年龄不在 30 到 40 之间的员工，你可以这样写：

SELECT * 
FROM Employees 
WHERE NOT (Age >= 30 AND Age <= 40);
1
2
3

请注意，WHERE 子句可以与 SQL 的许多其他子句（如 FROM、GROUP BY、HAVING、ORDER BY 等）一起使用，以创建更复杂的查询。

# SQL AND & OR

在 SQL 中，AND 和 OR 是逻辑运算符，用于组合或修改 WHERE 子句中的条件。这些运算符允许你在查询中设置多个条件。

AND 运算符表示所有条件必须都为真，才能返回行。以下是 AND 运算符的一个示例：

SELECT column1, column2, ...
FROM table_name
WHERE condition1 AND condition2 ... AND conditionN;
1
2
3

例如，如果我们有一个 "Employees" 表，并且我们希望找到年龄超过 30 岁并且职位是 "Manager" 的员工，我们可以这样做：

SELECT * 
FROM Employees 
WHERE Age > 30 AND Position = 'Manager';
1
2
3

OR 运算符表示只要任何一个条件为真，就会返回行。以下是 OR 运算符的一个示例：

SELECT column1, column2, ...
FROM table_name
WHERE condition1 OR condition2 ... OR conditionN;
1
2
3

例如，如果我们希望找到年龄超过 30 岁或者职位是 "Manager" 的员工，我们可以这样做：

SELECT * 
FROM Employees 
WHERE Age > 30 OR Position = 'Manager';
1
2
3

你还可以使用圆括号组合 AND 和 OR 条件，来构建更复杂的查询。例如，假设我们想找到所有年龄超过 30 岁，并且职位是 "Manager" 或 "Engineer" 的员工，我们可以这样做：

SELECT * 
FROM Employees 
WHERE Age > 30 AND (Position = 'Manager' OR Position = 'Engineer');
1
2
3

在这个例子中，圆括号确保了 "Manager" 或 "Engineer" 的条件只影响职位，而不影响年龄。

# SQL ORDER BY

SQL 中的 ORDER BY 关键字用于对查询结果进行排序。你可以按照升序（默认）或降序排序。

基本语法如下：

SELECT column1, column2, ...
FROM table_name
ORDER BY column1 [ASC|DESC], column2 [ASC|DESC], ...;
1
2
3

在这里：

  • ASC 表示升序（默认）。
  • DESC 表示降序。

你可以根据一个或多个列进行排序。当你根据多个列进行排序时，结果首先根据第一列进行排序，然后根据第二列进行排序，依此类推。

举例来说，假设我们有一个名为 "Employees" 的表，我们可以按照姓名（Name）升序排序，如下：

SELECT * 
FROM Employees 
ORDER BY Name ASC;
1
2
3

如果我们想按照薪水（Salary）降序排序，可以这样做：

SELECT * 
FROM Employees 
ORDER BY Salary DESC;
1
2
3

如果我们想先按部门（Department）升序排序，然后在每个部门中按薪水降序排序，可以这样做：

SELECT * 
FROM Employees 
ORDER BY Department ASC, Salary DESC;
1
2
3

# SQL GROUP BY

GROUP BY 是 SQL 中的一个关键字，用于将查询结果集中的行组合到一起，以便能够对它们应用聚合函数，例如 COUNT(), SUM(), AVG(), MAX() 或 MIN()。

GROUP BY 语句的基本语法如下：

SELECT column1, column2, ..., aggregate_function(column)
FROM table_name
WHERE condition
GROUP BY column1, column2, ...;
1
2
3
4

这里的 aggregate_function 是指聚合函数，如 COUNT(), SUM(), AVG(), MAX() 或 MIN()。

例如，假设我们有一个 "Sales" 表格，其中包含 "Item", "Quantity" 和 "Price" 列。如果我们想知道每个商品的总销售额，我们可以使用 GROUP BY 和 SUM()，如下：

SELECT Item, SUM(Quantity * Price) as TotalSales
FROM Sales
GROUP BY Item;
1
2
3

这将返回一个结果集，其中每一行对应一种商品，"TotalSales" 列显示的是该商品的总销售额。

我们还可以结合 WHERE 语句使用 GROUP BY 来只包括满足特定条件的行。例如，以下查询将只计算 2023 年的销售额：

SELECT Item, SUM(Quantity * Price) as TotalSales
FROM Sales
WHERE Year = 2023
GROUP BY Item;
1
2
3
4

此外，GROUP BY 可以根据多个列进行分组。例如，如果我们想按年份和商品分类统计销售额，我们可以这样写：

SELECT Year, Item, SUM(Quantity * Price) as TotalSales
FROM Sales
GROUP BY Year, Item;
1
2
3

在这个查询中，结果将首先根据年份进行分组，然后在每个年份中根据商品进行分组。

# SQL INSERT INTO

INSERT INTO 是 SQL 中的一个命令，用于向数据库表中插入新的行（记录）。基本语法有两种形式：

  1. 插入完整的行：
INSERT INTO table_name
VALUES (value1, value2, value3, ...);
1
2

在这种形式中，必须为表中的每一列提供一个值，并且这些值的顺序必须与表中列的顺序一致。

  1. 插入特定列的值：
INSERT INTO table_name (column1, column2, column3, ...)
VALUES (value1, value2, value3, ...);
1
2

在这种形式中，只为特定列插入值。这些列和值的顺序必须匹配。

例如，如果你有一个名为 "Students" 的表，它有 "StudentID", "LastName", "FirstName" 和 "Major" 列，你可以使用以下命令插入一行数据：

INSERT INTO Students
VALUES (1, 'Smith', 'John', 'Computer Science');
1
2

或者，如果你只想插入 "StudentID" 和 "LastName" 的值，你可以使用以下命令：

INSERT INTO Students (StudentID, LastName)
VALUES (2, 'Johnson');
1
2

在第二个例子中，因为 "FirstName" 和 "Major" 没有指定值，所以它们将被设置为 NULL（除非有默认值）。

# SQL UPDATE

UPDATE 是 SQL 中的一个关键字，用于修改已存在的记录在数据库表中的数据。

UPDATE 语句的基本语法如下：

UPDATE table_name
SET column1 = value1, column2 = value2, ...
WHERE condition;
1
2
3

在 SET 子句之后，你需要列出你想要更改的列名和它们的新值。你可以一次更新一个或多个列。

WHERE 子句是可选的，但是在大多数情况下，你都应该使用它。如果你省略 WHERE 子句，UPDATE 语句将更新表中的所有行。这可能会导致数据丢失，因为你无法再将数据恢复到原来的状态。

例如，假设我们有一个名为 "Employees" 的表，其中包含 "EmployeeID", "LastName", "FirstName" 和 "Salary" 列。如果我们想为 EmployeeID 为 1 的员工涨薪，我们可以这样写 UPDATE 语句：

UPDATE Employees
SET Salary = Salary + 500
WHERE EmployeeID = 1;
1
2
3

这将把 EmployeeID 为 1 的员工的 Salary 列的值增加 500。

请注意，更新操作是不可逆的，因此在更新数据库之前，务必确保备份你的数据或者你的 SQL 查询是正确的。

# SQL DELETE

DELETE 是 SQL 中的一个关键字，用于删除数据库表中的行。

DELETE 语句的基本语法如下：

DELETE FROM table_name
WHERE condition;
1
2

你可以使用 WHERE 子句来指定你要删除哪些行。如果省略 WHERE 子句，DELETE 语句将删除表中的所有行。

例如，假设我们有一个名为 "Employees" 的表，其中包含 "EmployeeID", "LastName", "FirstName" 和 "Salary" 列。如果我们想删除 EmployeeID 为 1 的员工的记录，我们可以这样写 DELETE 语句：

DELETE FROM Employees
WHERE EmployeeID = 1;
1
2

这将删除 EmployeeID 为 1 的员工的所有信息。

请注意，DELETE 语句会永久删除数据，这个操作是不可逆的。因此，确保在删除数据之前，你已经备份了所有重要数据，并且你的 SQL 查询是正确的。除非你确定要删除表中的所有数据，否则不要省略 WHERE 子句。
