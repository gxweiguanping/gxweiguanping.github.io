JDK1.8新特性

# JDK1.8新特性

# JDK1.8都有哪些新特性？

  • 接口的默认方法&静态方法 ：在Java 8中，接口可以包含默认方法和静态方法。默认方法是接口中的一个具体实现，可以在接口中直接定义方法的实现。静态方法是接口中的一个静态方法，可以直接通过接口名调用。

    public interface MyInterface {
    // 静态方法
    static void staticMethod() {
        System.out.println("接口中可以存在静态方法的实现，可以直接通过接口名调用");
    }

    // 默认方法
    default void defaultMethod(){
        System.out.println("接口中可以存在默认方法的实现");
    }
     // 抽象方法
    void abstractMethod();
}

// 实现接口
class MyClass implements MyInterface {
    // 实现抽象方法
    @Override
    public void abstractMethod() {
        System.out.println("MyClass 实现的抽象方法");
    }
}

public class Main {
    public static void main(String[] args) {
        // 调用默认方法
        MyInterface myObj = new MyClass();
        myObj.defaultMethod();

        // 调用静态方法
        MyInterface.staticMethod();

        // 调用抽象方法
        myObj.abstractMethod();
    }
}

  • Lambda 表达式&函数式接口 ：Lambda 表达式本质上是一段匿名内部类，也可以是一段可以传递的代码。Lambda 允许把函数作为一个方法的参数（函数作为参数传递到方法中），使用 Lambda 表达式使代码更加简洁，但是也不要滥用，否则会有可读性等问题，《Effective Java》作者 Josh Bloch 建议使用Lambda 表达式最好不要超过3行。

  • Stream API ：Stream API提供了一种更简洁、更易于并行处理数据集合的方式。通过Stream API，我们可以对集合进行过滤、映射、排序、聚合等操作，提高了代码的可读性和简洁性。

  • 新的日期和时间API ：Java 8引入了新的日期和时间API（java.time包），提供了更好的日期和时间处理方式。新的API解决了旧的Date和Calendar类在处理日期和时间时的一些问题，并提供了更多的功能和灵活性。

  • Optional 类 ：Optional类是一个容器类，用于表示一个值可能存在或不存在。通过Optional类，我们可以避免空指针异常，并更好地处理可能为空的值。

    import java.util.Optional;

public class Main {
    public static void main(String[] args) {
        String str = "Hello, world!";
        
        // 创建一个包含可能为 null 的值的 Optional 对象
        Optional<String> optionalStr = Optional.ofNullable(str);
        
        // 判断值是否存在
        if (optionalStr.isPresent()) {
            // 如果值存在，打印值
            System.out.println("Value is present: " + optionalStr.get());
        } else {
            // 如果值不存在，打印提示信息
            System.out.println("Value is absent.");
        }
        
        // 使用 orElse 方法提供默认值
        String defaultValue = optionalStr.orElse("Default Value");
        System.out.println("Default Value: " + defaultValue);
        
        // 使用 ifPresent 方法在值存在时执行操作
        optionalStr.ifPresent(value -> System.out.println("Length of value: " + value.length()));
        
        // 使用 map 方法转换值
        Optional<Integer> lengthOptional = optionalStr.map(String::length);
        lengthOptional.ifPresent(length -> System.out.println("Length of value: " + length));
    }
}

# Lambda 表达式了解多少？

Lambda 表达式本质上是一段匿名内部类，也可以是一段可以传递的代码。

比如我们以前使用Runnable创建并运行线程：

public class Main {
    public static void main(String[] args) {
      new Thread(new Runnable() {
          @Override
          public void run() {
              System.out.println("这里是我们要做的事情");
          }
      });
    }
}

当然不是每个接口都可以缩写成 Lambda 表达式。只有那些函数式接口（Functional Interface）才能缩写成 Lambda 表示式。

所谓函数式接口（Functional Interface）就是只包含一个抽象方法的声明。针对该接口类型的所有 Lambda 表达式都会与这个抽象方法匹配。

@FunctionalInterface
public interface Runnable {
    public abstract void run();
}

# Java8有哪些内置函数式接口？

JDK 1.8 API 包含了很多内置的函数式接口。其中就包括我们在老版本中经常见到的Comparator 和 Runnable，Java 8 为他们都添加了 @FunctionalInterface 注解，以用来支持 Lambda 表达式。

除了这两个之外，还有Callable、Predicate、Function、Supplier、Consumer等等。

# Optional了解吗？

Optional 是用于防范 NullPointerException 。

可以将 Optional 看做是包装对象（可能是 null , 也有可能非 null ）的容器。当我们定义了 一个方法，这个方法返回的对象可能是空，也有可能非空的时候，我们就可以考虑用 Optional 来包装它，这也是在 Java 8 被推荐使用的做法。

public class Main {
    public static void main(String[] args) {

        List<String> arr = null;
        // 不判空直接获取，有可能arr是null，就空指针异常了
        // arr.forEach(a -> System.out.println(a));
        List<String> newArr = Optional.ofNullable(arr).orElse(new ArrayList<>());
        newArr.forEach(a-> System.out.println(a));
    }
}

# Stream 流用过吗？

Stream 流，简单来说，使用 java.util.Stream 对一个包含一个或多个元素的集合做各种操作。这些操作可能是 中间操作 亦或是 终端操作。 终端操作会返回一个结果，而中间操作会返回一个 Stream 流。

Stream流一般用于集合，我们对一个集合做几个常见操作：

public class Student {
    private String name;
    private Integer age;
    private Double score;

    public Student(String name, Integer age, Double score) {
        this.name = name;
        this.age = age;
        this.score = score;
    }
    // 省略get、set、toString方法
}

public class Main {
    public static void main(String[] args) {
        List<Student> studentList = new ArrayList<>();
        studentList.add(new Student("小明", 15, 89.22));
        studentList.add(new Student("小红", 16, 78.50));
        studentList.add(new Student("小华", 14, 92.80));
        studentList.add(new Student("小李", 17, 85.60));

        // 过滤
        studentList.stream()
                .filter(student -> student.getAge() > 15)
                .forEach(System.out::println);

        // 排序
        studentList.stream()
                .sorted((o1, o2) -> o1.getAge() - o2.getAge())
                .filter(student -> student.getAge() > 15)
                .forEach(System.out::println);

        // 转成map，(v1, v2) -> v1)表示key重复的时候，已v1为准，v2会被覆盖
        studentList.stream()
                .filter(student -> student.getName() != null && student.getScore() != null)
                .collect(Collectors.toMap(Student::getName, Student::getAge, (v1, v2) -> v1));

        // 分组
        Map<String, List<Student>> studentMap = studentList.stream()
            .collect(Collectors.groupingBy(Student::getName));

        // 计数
        studentList.stream()
                .filter(student -> student.getAge() > 15)
                .count();

        // 计算分数总和
        studentList.stream()
                .mapToDouble(Student::getScore)
                .sum();
    }
}
