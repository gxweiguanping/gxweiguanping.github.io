Diamond语法，也就是在Java 7中引入的类型推断特性，允许开发者在使用泛型时不必在构造器的右边重复泛型的类型。

例如，在Spring 5.0之前，可能需要这样声明一个带泛型的集合：

List<String> list = new ArrayList<String>();
1

而有了Diamond语法后，你可以省略右边的泛型类型，让编译器自动推断出来：

List<String> list = new ArrayList<>();
1

# 可重复Annotations(JSR337)

JSR 337 是Java Platform Standard Edition 8的规范请求（Java Specification Request），其中包含了可重复注解的特性。

在Spring框架中，这个特性自 4.0版本 开始得到支持。Spring 4.0是在2013年发布的，它包括对Java 8的许多新特性的支持，尽管Java 8在2014年才正式发布。

@PropertySources 是Spring框架中的一个注解，它是一个容器注解，用于聚合多个@PropertySource注解。@PropertySource注解用于声明属性源，属性源可以是外部配置文件，这些文件中包含了应用程序运行所需的各种环境参数或配置项。

在Spring 4.0之前，如果想要添加多个@PropertySource注解到一个类上，需要使用@PropertySources这个容器注解。从Spring 4.0开始，由于支持了可重复注解，可以直接在类上多次使用@PropertySource注解，而不需要再使用@PropertySources容器注解。

例如，在Spring 4.0之前，可能需要这样使用：

@Configuration
@PropertySources({
    @PropertySource("classpath:foo.properties"),
    @PropertySource("classpath:bar.properties")
})
public class AppConfig {
    // ...
}

# Java EE Web技术相关

JSR 是 Java Specification Request 的缩写，它是 Java 社区过程（Java Community Process，JCP）的一部分，用于提出和规定 Java 平台的各种标准和增强。JSR 是一个正式的文档，它描述了所提议的规范和其技术规格。每个 JSR 都通过一系列的审查和社区投票过程，最终可能被批准并成为 Java 平台的官方部分。

JSR 涵盖了 Java 平台的各个方面，包括：

  • 核心 Java 语言的改进
  • 标准 Java 类库的扩展
  • Java 企业版（Java EE）的各种组件和 API
  • Java 微型版（Java ME）的特性
  • Java 虚拟机（JVM）的增强

每个 JSR 都由一个规范领导者负责，通常是来自相关领域的专家或组织。规范领导者负责指导 JSR 的发展，并与专家组（Expert Group, EG）合作，后者由对该领域感兴趣的其他专家和组织成员组成。专家组成员共同工作，提供反馈，讨论规范，并最终帮助形成最终的规范文档。
