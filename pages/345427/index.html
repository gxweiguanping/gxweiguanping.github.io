# 安装

# 传统方式安装

1、root状态下 进入local目录

cd /usr/local

# 新建influxDB目录
mkdir influxDB
1
2
3
4

2、下载influxdb安装包（使用的是1.8.10版本）

wget https://dl.influxdata.com/influxdb/releases/influxdb-1.8.10_linux_amd64.tar.gz
1

3、解压下载好的influxdb-1.81.0_linux_amd64.tar.gz

tar -zxvf influxdb-1.8.10_linux_amd64.tar.gz

# 修改解压好的目录名，我这里解压好没有修改的目录名是influxdb-1.8.0-1, 修改后influxdb-1.8.10
mv influxdb-1.8.10-1 influxdb-1.8.10
1

3
4

4、进入修改好的influxdb-1.8里的bin目录;

cd influxdb-1.8.10/usr/bin
1

5、书写启动脚本

# 书写启动脚本(用于: 后台启动脚本)
vi startup.sh

# 输入内容如下(注意这也是内容#!/bin/bash)，保存退出
#!/bin/bash
nohup /usr/local/influxdb-1.8.10/usr/bin/influxd -config \
/usr/local/influxdb-1.8.10/etc/influxdb/influxdb.conf \
&>/home/influxdb/log/influxdb.log &

#保存退出然后授执行权
chmod +x startup.sh

目录说明

  • etc主要用来存储influxdb系统配置信息
  • usr主要用来存储influxdb操作相关脚本文件
  • var主要用来存储influxdb运行日志、以及产生数据和依赖库文件

# docker方式安装

1、在Docker库中查找influxDB镜像

# 在Docker库中查找influxDB镜像文件
docker search influxdb
1
2

也可以在dockerHub (opens new window)上查找

2、从Docker库中拉取influxDB镜像

# 从docker库中拉取influxDB特定版本的镜像，默认拉取最新版本
docker pull influxdb:1.8.10		

# 查看本地已有的镜像文件
docker images
1
2
3
4
5

3、启动influxDB容器

docker run -d -p 8073:8083 -p8076:8086 \
     -v /data/backups/influxdb/data:/var/lib/influxdb \
     -v /data/backups/influxdb/influxdb.conf:/etc/influxdb/influxdb.conf \
     --name influx1.8.10 \
     --restart=always \
     influxdb:1.8.10
1
2
3
4
5
6
  • -d: 这个选项告诉 Docker 在后台运行容器，也就是以分离模式（detached mode）运行。
  • -p 8073:8083 -p 8076:8086: 这两个选项将容器内部的端口映射到主机上的端口。具体来说，将容器内的 8083 端口映射到主机的 8073 端口，将容器内的 8086 端口映射到主机的 8076 端口。这允许你通过主机上的这些端口来访问 InfluxDB 的 Web 界面（8083）和 HTTP API（8086）。
  • -v /data/backups/influxdb/data:/var/lib/influxdb: 这个选项用于将主机的目录 /data/backups/influxdb/data 挂载到容器内的 /var/lib/influxdb 目录。这是为了在容器内保存 InfluxDB 数据，使数据持久化。
  • -v /data/backups/influxdb/influxdb.conf:/etc/influxdb/influxdb.conf：将配置文件挂在到主机
  • --name influx1.8.10: 这个选项为容器指定一个名称，即 "influx1.8.10"。
  • --restart=always: 这个选项告诉 Docker，如果容器停止或宕机，它应该自动重新启动。
  • influxdb:1.8.10: 这是要运行的 Docker 镜像的名称和版本。在这里，你正在运行 InfluxDB 版本 1.8.10 的官方镜像。

# 客户端安装

DBeaver 是一个通用的数据库管理工具和 SQL 客户端，支持 MySQL，PostgreSQL，Oracle，DB2，MSSQL，Sybase, Mimer, HSQLDB，Derby，InfluxDB以及其他兼容 JDBC 的数据库。DBeaver 提供一个图形界面用来查看数据库结构、执行SQL查询和脚本，浏览和导出数据，处理BLOB/CLOB 数据，修改数据库结构等等。

# 激活 DBeaver

本激活方式支持 DBeaver 全系列 Lite、Enterprise、Ultimate 目前最新 22.0 版本，亲测可用~

DBeaver 运行需要使用 Java 环境。不要使用 DBeaver 自带的 jre 。使用本激活包需要删除 DBeaver 目录下自带的 jre，自行安装JDK 11。

修改 DBeaver 目录下的 dbeaver.ini 以下配置：

替换 dbeaver.ini 内由 -vm 指定的 jdk 11 的 bin 路径，把地址换成自己安装的！

如果你的 dbeaver.ini 内没有 -vm 参数，请在 首行 添加你安装 jdk 的安装路径：

如果你配置了 jdk 11 的环境变量，该参数可以不用配置。

-vm
D:\java\jdk\jdk-11.0.19\bin
1
2

接下来添加激活包配置：

在 dbeaver.ini 编辑窗口末行添加：-javaagent:/absolute/path/to/dbeaver-agent.jar

示例:

mac:      -javaagent:/Users/dbeaver-agent/dbeaver-agent.jar
linux:    -javaagent:/home/dbeaver-agent/dbeaver-agent.jar
windows:  -javaagent:D:\software\dbeaver-agent\dbeaver-agent\dbeaver-agent.jar
