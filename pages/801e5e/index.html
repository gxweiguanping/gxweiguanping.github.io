JVM监控及诊断工具-命令行篇

# JVM监控及诊断工具-命令行篇

# 概述

性能诊断是软件工程师在日常工作中需要经常面对和解决的问题，在用户体验至上的今天，解决好应用的性能问题能带来非常大的收益。

Java 作为最流行的编程语言之一，其应用性能诊断一直受到业界广泛关注。可能造成 Java 应用出现性能问题的因素非常多，例如线程控制、磁盘读写、数据库访问、网络I/O、垃圾收集等。想要定位这些问题，一款优秀的性能诊断工具必不可少。

体会1：使用数据说明问题，使用知识分析问题，使用工具处理问题。

体会2：无监控、不调优！

简单命令行工具

在我们刚接触java学习的时候，大家肯定最先了解的两个命令就是javac，java，那么除此之外，还有没有其他的命令可以供我们使用呢？

我们进入到安装jdk的bin目录，发现还有一系列辅助工具。这些辅助工具用来获取目标 JVM 不同方面、不同层次的信息，帮助开发人员很好地解决Java应用程序的一些疑难杂症。

官方源码地址：http://hg.openjdk.java.net/jdk/jdk11/file/1ddf9a99e4ad/src/jdk.jcmd/share/classes/sun/tools

# java：执行 Java 程序

java 命令用于在 Java 虚拟机（JVM）上执行 Java 程序。它接受一个类名作为参数，JVM 会加载该类并执行它的 main 方法。这是运行 Java 应用程序的命令。例如：

java HelloWorld
1

java 命令可选参数：

参数 作用
-classpath <path> 指定查找用户类文件和资源文件的位置
-cp <path> 简写形式，指定查找用户类文件和资源文件的位置
-D<property>=<value> 设置系统属性
-verbose[:class|gc|jni] 显示详细的类加载、垃圾回收或JNI调用信息
-version 显示版本信息
-showversion 显示版本信息并继续执行
-help 显示帮助信息
-X 显示非标准选项
-ea[:<packagename>... :<classname>]
-da[:<packagename>... :<classname>]
-esa 启用系统断言
-dsa 禁用系统断言
-jar <jarfile> [args...] 执行包含主类的JAR文件
-Xmx 设置 JVM 最大堆内存大小
-Xms 设置 JVM 初始堆内存大小

# javac：将 Java 源代码文件编译成字节码文件

javac 命令用于将 Java 源代码文件（以 .java 扩展名结尾）编译成字节码文件（以 .class 扩展名结尾）。编译后的字节码可以由 JVM 执行。例如：

javac HelloWorld.java
1
参数 作用
-classpath <path> 指定查找用户类文件和注释处理程序的位置
-cp <path> 简写形式，指定查找用户类文件和注释处理程序的位置
-sourcepath <path> 指定查找源文件的位置
-d <directory> 指定生成的类文件的输出目录
-encoding <encoding> 指定源文件的字符编码
-g 生成所有调试信息
-g:none 不生成任何调试信息
-nowarn 不显示任何警告
-verbose 显示编译器的详细输出
-version 显示版本信息
-help 显示帮助信息

# javap：反汇编编译后的字节码文件

参数 作用
-c 显示反编译后的字节码指令
-s 显示内部类型的签名
-l 显示行号和本地变量表
-verbose 显示附加信息，包括版本号、常量池、访问标志等
-classpath <path> 指定查找用户类文件和注释处理程序的位置
-bootclasspath<path> 覆盖引导类文件的位置
-p 显示私有成员
-protected 显示受保护成员
-public 显示公共成员
-package 显示默认访问级别成员
-version 显示版本信息
-help 显示帮助信息

javap 命令是 Java 反汇编工具，用于反汇编编译后的字节码文件，以可读的方式显示类的详细信息，包括成员变量、方法、访问修饰符等。这对于了解代码的底层实现和优化很有帮助。例如：

javap -c HelloWorld.class
1

这将显示 HelloWorld.class 字节码文件的详细信息，包括字节码指令。

javap 命令可选参数：

  • -c：显示字节码指令。
  • -s：显示类的签名。
  • -l：显示行号和本地变量表。
  • -verbose：显示附加详细信息。
  • -version：显示工具版本信息。

# jps：查看正在运行的Java进程

jps（Java Virtual Machine Process Status Tool）是Java开发工具包（JDK）中的一个命令行工具，用于查看正在运行的Java进程。它可以显示Java进程的进程ID（PID）和主类名。

参数 作用
-q 只显示进程ID，不显示类名、JAR文件名和传递给main方法的参数
-m 显示传递给main方法的参数
-l 显示完整的包名、应用程序主类名或JAR文件名
-v 显示传递给JVM的参数
-V 输出JVM的参数
-J<flag> 传递<flag>给JVM
-F 输出完整的类名，包括JAR文件名或应用程序主类名
-m 输出传递给main方法的参数
-h 显示帮助信息

使用jps命令可以方便地查看正在运行的Java进程，以及它们的PID和主类名。以下是jps命令的使用示例：

查看所有正在运行的Java进程：

jps
1

输出示例：

1234 MainClass1
5678 MainClass2
1
2

查看Java进程的详细信息（包括JVM参数）：

jps -lvm
1

输出示例：

1234 MainClass1 -Djava.library.path=/path/to/libs -Xmx512m
5678 MainClass2 -Djava.library.path=/path/to/libs -Xmx1024m
1
2

查看指定用户的Java进程：

jps -l -u username
1

输出示例：

1234 MainClass1
5678 MainClass2
1
2

查看远程主机上的Java进程：

jps -l -m -v -remoteprocess hostname
1

输出示例：

1234 MainClass1 -Djava.library.path=/path/to/libs -Xmx512m
5678 MainClass2 -Djava.library.path=/path/to/libs -Xmx1024m
1
2

jps命令对于监控和管理Java进程非常有用，可以用于识别正在运行的Java应用程序以及它们的PID和主类名。

# jinfo：实时查看和修改JVM配置参数

用于实时查看和修改Java虚拟机（JVM）的配置参数。它可以显示JVM的配置参数、系统属性和环境变量，并且可以在运行时修改这些参数。

参数 作用
-flag <name> 查看指定名称的JVM标志参数的值
-flag [+\|-]<name> 在这个参数中，<name> 是要操作的JVM标志参数的名称。而 [+|-] 则表示对该参数进行启用或禁用的操作,当使用 + 时，表示启用指定的JVM标志参数。当使用 - 时，表示禁用指定的JVM标志参数。
-flag <name>=<value> 设置指定名称的JVM标志参数的值
-sysprops 显示JVM的系统属性
-sysprops <pid> 显示指定进程ID（PID）的JVM的系统属性
-env 显示JVM的环境变量
-env <pid> 显示指定进程ID（PID）的JVM的环境变量
-h 显示帮助信息

使用jinfo命令可以方便地查看和修改JVM的配置参数。以下是jinfo命令的使用示例：

查看JVM的配置参数、系统属性和环境变量：

jinfo <pid>
1

其中，<pid>是Java进程的进程ID。

示例：

jinfo 12345
1

这将显示进程ID为12345的Java进程的配置参数、系统属性和环境变量。

查看JVM的配置参数：

jinfo -flags <pid>
1

这将显示进程ID为<pid>的Java进程的配置参数。

示例：

jinfo -flags 12345
1

这将显示进程ID为12345的Java进程的配置参数。

修改JVM的配置参数：

jinfo -flag <name>=<value> <pid>
1

这将进程ID为<pid>的Java进程的配置参数<name>的值修改为<value>。

示例：

jinfo -flag MaxHeapSize=512m 12345
1

这将将进程ID为12345的Java进程的最大堆内存大小（MaxHeapSize）修改为512MB。

注意

jinfo命令只能修改可被动态修改的JVM配置参数，而不能修改不可被动态修改的参数。此外，修改参数可能会对应用程序的性能和稳定性产生影响，因此在修改参数之前请谨慎考虑。
