# 安装redis

# 步骤1：拉取 Redis 镜像

Docker 镜像是包含了应用程序和它所需要环境的可执行包。在这个步骤中，我们将从 Docker Hub（Docker 的公共镜像库）中拉取 Redis 镜像。在终端中输入以下命令：

docker pull redis
1

# 步骤2：创建并运行 Redis 容器

现在，你可以使用刚才下载的 Redis 镜像来创建一个新的 Docker 容器，并运行它。使用以下命令：

docker run -it -d --name redis -p6379:6379 redis --bind 0.0.0.0 --protected-mode no  --requirepass 1qaz@2wsx
1
  • docker run: 这是 Docker 命令行接口的一个基本命令，用于启动新的 Docker 容器。
  • -it: 这两个标志的结合在 Docker 中通常用于打开一个新的终端会话，并且能够接收用户的输入。-i（或 --interactive）保证了容器的 STDIN 是开启的，-t（或 --tty）则为 Docker 分配一个伪终端或终端。
  • -d: 这个标志表示 "detached" 模式，这意味着 Docker 容器将在后台运行。
  • --name redis: 这个参数为 Docker 容器指定了一个名字，这样你就可以用 docker stop redis 这样的命令来停止容器，而不是使用容器 ID。
  • -p 6379:6379: 这个参数指示 Docker 将主机的 6379 端口映射到 Docker 容器的 6379 端口，这样你就可以通过主机的 6379 端口访问到 Docker 容器内运行的 Redis 服务器。
  • redis: 这是 Docker 镜像的名字，也就是我们要运行的应用。
  • --bind 0.0.0.0: 这个参数是 Redis 的配置选项，它告诉 Redis 绑定到所有可用的 IP 地址，使得从任何地方都可以连接到 Redis。
  • --protected-mode no: 这个参数也是 Redis 的配置选项，它关闭了 Redis 的保护模式。在保护模式下，Redis 默认只接受来自本地的连接。关闭保护模式可以使得从任何地方都可以连接到 Redis，但是需要注意的是这可能带来安全风险。
  • --requirepass 1qaz@2wsx: 这个参数是 Redis 的配置选项，它设置了 Redis 的密码为 1qaz@2wsx。设置了密码后，客户端在执行任何其他命令之前，需要先通过 AUTH <password> 命令验证密码。

# 步骤3：验证 Redis 安装

要确认 Redis 容器是否正确运行，你可以使用 Docker 的 exec 命令来在运行中的容器内执行命令。以下命令将在 myredis 容器内启动一个 Redis CLI（命令行接口）会话：

docker exec -it redis redis-cli
1

现在，你可以在 Redis CLI 中输入 Redis 命令来测试服务。例如，尝试使用 ping 命令：

ping
1

如果 Redis 正在运行，它将返回 PONG。
