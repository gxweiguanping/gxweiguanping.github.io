# 相关概念

# 概念

  • database：数据库是 InfluxDB 中的顶层容器，用于隔离不同的应用或数据集。每个数据库可以包含多个测量（measurements）。
  • measurement：测量是 InfluxDB 中的概念，类似于关系型数据库中的表（table），每个测量通常代表一个特定类型的时间序列数据，例如温度、湿度、CPU使用率等。
  • points：数据点，数据点是测量中的一行数据。每个数据点包括三个关键组成部分：
    • 时间戳（Time）：表示数据点的时间。时间戳是 InfluxDB 中的核心概念，因为时间在时间序列数据中非常重要。时间通常以纳秒级精度表示。
    • 数据（Field）：这是实际的数据值。它可以是浮点数、整数、字符串等。每个数据点可以包含一个或多个字段，以便存储不同类型的数据。不经常查询数据可以直接存储为field。
    • 标签（Tags）：标签是用于标识数据的元数据。它们是键值对，用于描述数据点。标签可以帮助你对数据进行过滤和分组。与字段不同，标签通常是字符串类型。主要是用来提高查询效率。

# 与Mysql概念对比

概念 MySQL InfluxDB
数据库（同） database database
表（不同） table measurement（测量）
列（不同） column tag（带索引，非必须）、field（不带索引）、timestemp（唯一主键）

Point相当于传统数据库里的一行数据，如下表所示：

Point属性 传统数据库中的概念
time（时间戳） 每个数据记录时间，是数据库中的主索引(会自动生成)
fields（字段数据） 各种记录值（没有索引的属性）也就是记录的值：温度，湿度
tags（标签） 各种有索引的属性：地区，海拔

注意

在influxdb中，字段必须存在。因为字段是没有索引的。如果使用字段作为查询条件，会扫描符合查询条件的所有字段值，性能不及tag。类比一下，fields相当于SQL的没有索引的列。tags是可选的，但是强烈建议你用上它，因为tag是有索引的,tags相当于SQL中的有索引的列。tag value只能是string类型。

# 类型说明

# tag 类型

tag都是 string 类型。

# field类型

filed支持四种类型int、float、string、boolean

类型 方式 示例
int 整数 42
float 浮点数 3.14
string 字符串 "Hello, World"
boolean 布尔值 true
