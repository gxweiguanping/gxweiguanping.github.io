# AOP

# 1.说说什么是AOP？

AOP（面向切面编程，Aspect-Oriented Programming）是一种编程范式。简单说，就是把一些业务逻辑中的相同的代码抽取到一个独 立的模块中，使它们独立于核心业务逻辑。

具体来说，假如我现在要crud写一堆业务，可是如何业务代码前后前后进行打印日志和参数的校验呢？

我们可以把 日志记录 和 数据校验 可重用的功能模块分离出来，然后在程序的执行的合适的地方动态地植入这些代码并执行。这样就简化了代码的书写。

AOP 可以将遍布应用各处的功能分离出来形成可重用的组件。在编译期间、装载期间或运行期间实现在不修改源代码的情况下给程序动态添加功能。从而实现对业务逻辑的隔离，提高代码的模块化能力。

AOP 的核心其实就是动态代理，如果是实现了接口的话就会使用 JDK 动态代理，否则使用 CGLIB 代理，主要应用于处理一些具有横切性质的系统级服务，如日志收集、事务管理、安全检查、缓存、对象池管理等。

# 2.AOP有哪些核心概念？

名称 说明
切面（Aspect） 切点和通知的结合，切面就是对横切关注点的抽象。
连接点（Joinpoint） 指那些被拦截到的点，在Spring 中，指可以被动态代理拦截目标类的方法。
切点（Pointcut） 指要对哪些Joinpoint进行拦截，即被拦截的连接点。
通知（Advice） 指拦截到Joinpoint之后要做的事情，即对切入点增强的内容。
目标（Targer） 指代理的目标对象
织入（Weaving） 指把增强代码应用到目标上，生成代理对象的过程。
代理（Proxy） 指生成的代理对象。
@Aspect
@Component
public class PermissionDataAspect {
    public PermissionDataAspect() {}

    @Pointcut("@annotation(com.demo.core.config.aspect.annotation.PermissionData)")
    public void pointCut() {}

    @Around("pointCut()")
    public Object arround(ProceedingJoinPoint point) throws Throwable {}
}

切面（Aspect）：PermissionDataAspect是一个切面（Aspect），负责切入到特定的连接点（Joinpoint），并定义了拦截规则和处理逻辑。

连接点（Joinpoint）：在代码执行过程中可以被拦截和增强的特定点。连接点是那些被@annotation(com.aiurt.core.config.aspect.annotation.PermissionData)注解标记的方法。

切入点（Pointcut）：是一组连接点的集合，定义了在哪里进行拦截。在上面的例子中，pointCut()方法就是定义了一个切入点，它拦截所有被@PermissionData注解标记的方法。

通知（Advice）：指定义在连接点上执行的操作。在上面的例子中，arround()方法是一个环绕通知（Around Advice），它在拦截到连接点后执行一些额外的操作，比如权限验证或者数据处理。

织入（Weaving）：是指把切面的逻辑织入到目标对象的过程。Spring AOP会在运行时生成代理对象，将切面的逻辑织入到这些代理对象中，以实现对连接点的拦截和处理。

# 3.AOP有哪些环绕方式？

AOP 一般有5 种环绕方式：

通知 对应注解 说明
before(前置通知) @Before 通知方法在目标方法调用之前执行
after (后置通知) @After 通知方法在目标方法调用之后或异常后调用
after-returning(返回后通知) @AfterReturning 通知方法会在目标方法执行返回后调用
after-throwing(抛出异常通知) @AfterThrowing 通知方法会在目标方法抛出异常后调用
around(环绕通知) @Around 通知方法会将目标方法封装起来

如果存在多个切面的情况下，可以通过 @Order 指定先后顺序，数字越小，优先级越高。

1）定义注解

@Retention(RetentionPolicy.RUNTIME)
@Target({ElementType.TYPE, ElementType.METHOD})
@Documented
public @interface PermissionData {
    String value() default "";
}

@Aspect
@Component
public class PermissionDataAspect {

    @Pointcut("@annotation(com.springboot.aop.aspect.annotation.PermissionData)")
    public void poincut() {

    }

    @Before("poincut()")
    public void before(){
        System.out.println("@Before通知执行................................");
    }

    @AfterReturning("poincut()")
    public void afterReturning() {
        System.out.println("@AfterReturning通知执行................................");
    }

    @Around("poincut()")
    public Object arround(ProceedingJoinPoint point) throws Throwable {
        System.out.println("@Around通知执行开始................................");
        //切点方法
        Object proceed = point.proceed();
        System.out.println("@Around通知执行结束................................");
        return proceed;
    }

    @AfterThrowing("poincut()")
    public void afterThrowing() {
        System.out.println("@AfterThrowing通知执行................................");
    }

    @After("poincut()")
    public void after() {
        System.out.println("@After通知执行................................");
    }
}

# 5.JDK 动态代理和 CGLIB 代理？

Spring的AOP是通过 动态代理 来实现的，动态代理主要有两种方式JDK动态代理和 CGLIB动态代理，这两种动态代理的使用和原理有些不同。

JDK 动态代理

  1. 如果你的目标类实现接口，那么Spring会选择使用用JDK动态代理来生成代理类。
  2. JDK会在运行时为目标类生成一个动态代理类$proxy*.class。
  3. 该代理类是实现了目标类接口，并且代理类会实现接口所有的方法增强代码。
  4. InvocationHandler ：InvocationHandler是一个接口，可以通过实现这个接口，定义横切逻辑，再通过反射机制（invoke）调用目标类的代码，在次过程中，可能包装逻辑，对目标方法进行前置后置处理。从而实现aop.

CgLib 动态代理

  1. 如果你的目标类没有实现接口，那么Spring会选择使用CGLIB动态代理来生成代理类。
  2. CgLib 动态代理是使用字节码处理框架 ASM ，其原理是通过字节码技术为一个类创建子类(还有其他类，主要为了增强调用时的效率)，并在子类中采用方法拦截的技术拦截所有父类方法的调用，顺势织入横切逻辑。
  3. CgLib 创建的动态代理对象性能比 JDK 创建的动态代理对象的性能高不少，但是 CGLib 在创建代理对象时所花费的时间却比 JDK 多得多，所以对于单例的对象，因为无需频繁创建对象，用 CGLib 合适，反之，使用 JDK 方式要更为合适一些。同时，由于 CGLib 由于是采用动态创建子类的方法，对于 final 方法，无法进行代理。
