# 基本操作

# 库 database

1、查看所有数据库

create database db_name
1

2、查看所有数据库

show databases
1

3、删除库

drop database db_name
1

4、使用某个数据库

use db_name
1

5、清除当前上下文库

clear database|db

# 表 measurements

1、查看所有的表

show measurements
1

2、删除一个表

注意

表不能显示创建,插入数据时自动插入表中数据

- drop measurement "test";

# 插入

语法：

insert <retention policy> measurement,tagKey=tagValue fieldKey=fieldValue timestamp
1

例如：

insert user,name=blr,phone=110 id=20,email="6099342@126.com" 1435362189575692182

从上面的输出，总结插入的语句写法:
1). insert + measurement + "," + tag=value,tag=value + + field=value,field=value2
2). tag与tag之间用逗号分隔,不能有多余的字符或空格
3). field与field之间用逗号分隔,不能有多余的字符或空格
4). tag与field之间用空格分隔,不能有多余的字符或空格
5). tag都是string类型，不需要引号将value包裹
6). field如果是string类型，需要加引号

# 普通查询

1、查询所有的 tag 和 field

SELECT * FROM test
1

2、根据name条件查询

SELECT * FROM test where "name"='xiaochen1'
1

注意

name是InfluxDB的关键字，所以查询时需要给name加上双引号区分

3、从单个measurement查询所有的field，不查tag

SELECT *::field FROM test
1

4、从单个measurement查询特定的field和tag

查询时至少要带上一个field key，如果只查询tag字段的话是查不到数据的

SELECT person_name,age FROM test
1

5、同时查询多张表

返回是将每张表不同的记录返回

select * from test,test1
1

6、模糊查询

前缀匹配 ，相当于 mysql 的 like 'abc%'

select * from test where "name"=~/^xiao/
1

后缀匹配 ，相当于 mysql 的 like '%abc'

select * from test where "name"=~/chen1$/
1

前后匹配，相当于 mysql 的 like '%abc%'

select * from test where "name"=~/chen/
1

# 聚合函数

注意

聚合函数只能对field字段进行操作，不能对tag字段操作，否则查询出来的列表是空的

如果我就要对tag字段进行聚合函数计算怎么办？ 其实可以通过子查询来实现：

select distinct(person_name) from (
   select * from test 
)
1
2
3

1、count() 统计

查询某个field字段的中的非空值数量

select count("age") from test
1

2、DISTINCT() 去重

select DISTINCT("age") from test
1

3、MEAN() 求平均值，这个平均值必须是数字类型

select MEAN("age") from test
1

4、MEDIAN() 求中位数，从单个字段（field）中的排序值返回中间值

select MEDIAN("age") from test
1

5、SPREAD() 返回字段的最小值和最大值之间的差值。

数据的类型必须是长整型或float64

select SPREAD("age") from test
1

6、SUM() 求和

select SUM("age") from test
1

7、BOTTOM() 返回一个字段中最小的N个值。

字段类型必须是长整型或float64类型。

select BOTTOM("age",6) from test
1

8、FIRST() 返回一个字段中最老的取值。

（第一个插入的数据），相当于select * from table order by asc limit 1

select FIRST("age") from test
1

9、LAST() 返回一个字段中最新的取值。

（最后一个插入的数据），相当于select * from table order by desclimit 1

select LAST("age") from test
1

10、MAX() 求最大值

select MAX("age") from test
1

11、PERCENTILE() 返回排序值排位为N的百分值。

字段的类型必须是长整型或float64

select PERCENTILE("age",3) from test
1

12、STDDEV() 返回一个字段中的值的标准偏差。

值的类型必须是长整型或float64类型。

select STDDEV("age") from test
1

# 分页查询

LIMIT 用法有2种

  • limit 10：查询前10条数据
  • limit size offset N： size表示每页 大小，N表示第几条记录开始查询

1、查询前10条数据

select * from test limit 10
1

2、分页，pageSize 为每页显示大小， pageIndex 为查询的页数

pageIndex = 1
pageSize = 10

# 公式
SELECT * FROM test LIMIT pageSize OFFSET (pageIndex-1)*pageSize
 
# 返回结果的前 10 行
SELECT * FROM your_measurement_name LIMIT 10 OFFSET 0

# in() 查询

关系型数据库可以用in关键字来查询多个值，就像这样

select * from sys_user where username in ('张三1','李四1','管理员');
1

但是时序数据库是没有 in 查询的，虽然in是保留的关键字，但是依然有办法解决, 还记得刚刚的模糊查询吗？

没错，就是用它来解决这个问题

同时匹配 123 和 thing ，因为前后都机上了 ^ 和 $ ，所以这已经不是模糊查询，而是完全匹配才会查询出来

select * from device_escalation_log where uuid =~/^123$|^thing$/
1

# 保留策略

influxDB是没有提供直接删除数据记录的方法，但是提供数据保存策略，主要用于指定数据保留时间，超过指定时间，就删除这部分数据。(数据库过期策略至少—个小时)，默认保存策略为永久保存，保存策略的应用范围是针对库来说的。

查看某个库的策略

show retention policies on "数据库名称"
1

查看当前库下的策略，需要先用 use database 命令指定库名

show retention policies
1

查询结果如下

> show retention policies on "db_name"
name    duration shardGroupDuration replicaN default
----    -------- ------------------ -------- -------
autogen 0s       168h0m0s           1        true
1
2
3
4
  • name: 策略名称
  • duration：数据保存时间，超过这个时间自动删除，0表示永久保存
  • shardGroupDuration：shardGroup的存储时间，分片组的持续时间为"168h0m0s"，即168小时。shardGroup是InfluxDB的一个基本储存结构，在这个时间内插入的数据查询较快，数据存放大于168小时查询速度降低；
    • Shard（分片）：分片是一种将时间序列数据分割成小块的方法。每个分片包含一段时间内的数据，例如一小时或一天的数据。
    • Shard Group（分片组）：分片组是一个集合，包含多个分片。它有一个特定的时间范围，称为 shardGroupDuration，通常是一周或一个月。在分片组内的所有分片都具有相同的 shardGroupDuration。
    • shardGroupDuration：这是分片组的持续时间。它规定了多长时间的数据将被存储在同一个分片组内。例如，如果 shardGroupDuration 设置为 7 天，那么一个分片组将包含最多 7 天的数据。
    • 底层原理：当新数据进入 InfluxDB，它会被放入当前时间范围内的分片组。一旦达到了 shardGroupDuration，分片组将被封存（不再接受新数据），然后一个新的分片组将创建用来接收新数据。旧的分片组将最终被压缩、存档或删除，以减小存储空间的使用。
  • replicaN：全称是REPLICATION，副本个数，副本数为1，表示每个数据点仅有一个副本。
  • default：是否默认策略，true表示autogen保留策略是默认的，这意味着新创建的测量（measurement）将使用这个保留策略。

# 创建数据保留策略

h（小时），d（天），w（星期）

CREATE RETENTION POLICY "保留策略名称" ON "数据库名称" DURATION "该保留策略对应的数据过期时间" REPLICATION "复制因子，开源的InfluxDB单机环境永远为1" SHARD DURATION "分片组的默认时长" DEFAULT
1

例子

create retention policy "testPolice" on myInfluxdb duration 72h  replication 1 SHARD DURATION 1h default
1

注意：duration过期时间必须大于等于shardGroupDuration：shardGroup时间

# 修改保留策略

alter retention policy "保留策略名称" on "数据库名称" duration 1d
1

修改默认保留策略

alter retention policy "保留策略名称" on "数据库名称" default
1

# 删除保留策略

drop retention policy "保留策略名称" on "数据库名称"
1
