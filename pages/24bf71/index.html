kafka集群安装

# kafka集群安装

# 前提准备

  • 需要提前安装好 docker 和 docker-compose
  • 使用的是wurstmeister/kafka镜像

# kafka常规安装

# 参数说明

参数 说明
KAFKA_BROKER_ID 0
KAFKA_NUM_PARTITIONS topic的分区数
KAFKA_DEFAULT_REPLICATION_FACTOR 分区的副本数
KAFKA_ZOOKEEPER_CONNECT zookeeper地址
KAFKA_LISTENERS 容器内访问的地址
KAFKA_ADVERTISED_LISTENERS 容器外访问的地址

# linux版本

docker命令方式和docker-compose方式二选一即可。

# docker-compose方式

version: '3'
services:
  zookeeper:
    image: wurstmeister/zookeeper
    container_name: zookeeper
    ports:
      - 2182:2181
    restart: always
    privileged: true
  
  kafka1:
    image: wurstmeister/kafka
    container_name: kafka1
    ports:
      - "9093:9092"
    environment:
      KAFKA_BROKER_ID: 1
      KAFKA_NUM_PARTITIONS: 3
      KAFKA_DEFAULT_REPLICATION_FACTOR: 2
      KAFKA_ZOOKEEPER_CONNECT: 192.168.1.174:2182
      KAFKA_LISTENERS: PLAINTEXT://0.0.0.0:9092
      KAFKA_ADVERTISED_LISTENERS: PLAINTEXT://192.168.1.174:9093
    volumes:
      - /data/docker-compose/kafka/broker1/logs:/opt/kafka/logs
      - /var/run/docker.sock:/var/run/docker.sock
    restart: always
    privileged: true
 
  kafka2:
    image: wurstmeister/kafka
    container_name: kafka2
    ports:
      - "9094:9092"
    environment:
      KAFKA_BROKER_ID: 2
      KAFKA_NUM_PARTITIONS: 3
      KAFKA_DEFAULT_REPLICATION_FACTOR: 2
      KAFKA_ZOOKEEPER_CONNECT: 192.168.1.174:2182
      KAFKA_LISTENERS: PLAINTEXT://0.0.0.0:9092
      KAFKA_ADVERTISED_LISTENERS: PLAINTEXT://192.168.1.174:9094
    volumes:
      - /data/docker-compose/kafka/broker2/logs:/opt/kafka/logs
      - /var/run/docker.sock:/var/run/docker.sock
    restart: always
    privileged: true
  
  kafka3:
    image: wurstmeister/kafka
    container_name: kafka3
    ports:
      - "9095:9092"
    environment:
      KAFKA_BROKER_ID: 3
      KAFKA_NUM_PARTITIONS: 3
      KAFKA_DEFAULT_REPLICATION_FACTOR: 2
      KAFKA_ZOOKEEPER_CONNECT: 192.168.1.174:2182
      KAFKA_LISTENERS: PLAINTEXT://0.0.0.0:9092
      KAFKA_ADVERTISED_LISTENERS: PLAINTEXT://192.168.1.174:9095
    volumes:
      - /data/docker-compose/kafka/broker3/logs:/opt/kafka/logs
      - /var/run/docker.sock:/var/run/docker.sock
    restart: always
    privileged: true

# docker命令方式

# ZooKeeper 服务
docker run -d --name zookeeper -p 2181:2181 --restart always zookeeper
1
# Kafka 服务

启动 Kafka Broker 1：

docker run -d \
  --name kafka1 \
  -p 9093:9092 \
  -e KAFKA_BROKER_ID=1 \
  -e KAFKA_NUM_PARTITIONS=3 \
  -e KAFKA_DEFAULT_REPLICATION_FACTOR=2 \
  -e KAFKA_ZOOKEEPER_CONNECT=192.168.1.174:2182 \
  -e KAFKA_LISTENERS=PLAINTEXT://0.0.0.0:9092 \
  -e KAFKA_ADVERTISED_LISTENERS=PLAINTEXT://192.168.1.174:9093 \
  -v /data/docker-compose/kafka/broker1/logs:/opt/kafka/logs \
  -v /var/run/docker.sock:/var/run/docker.sock \
  --restart always \
  wurstmeister/kafka

# 查看kafka版本

在Kafka的安装目录下，打开终端或命令提示符。 运行以下命令来查看Kafka的版本信息:

bin/ kafka-server-start.sh --version
1

输出（docker方式安装）

/opt/kafka_2.13-2.8.1/bin # sh ./kafka-server-start.sh --version
[2023-12-20 08:02:20,588] INFO Registered kafka:type=kafka.Log4jController MBean (kafka.utils.Log4jControllerRegistration$)
2.8.1 (Commit:839b886f9b732b15)
1
2
3

根据提供的输出信息，Kafka 的版本是 2.8.1，Commit ID 是 839b886f9b732b15。这个信息表明正在使用 Kafka 2.8.1 版本。
