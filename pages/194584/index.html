# SQL AVG() 函数

AVG() 是 SQL 中的一个聚合函数，它用于计算一列中数值的平均值。

基本语法如下：

SELECT AVG(column_name)
FROM table_name
WHERE condition;
1
2
3
  • column_name 是你要计算平均值的列的名字。
  • table_name 是你从中选择数据的表的名字。
  • WHERE condition 是一个可选的条件语句，用来限制哪些行应该被包括在计算中。

例如，如果你有一个叫做 Orders 的表，里面有一个叫做 Quantity 的列，你可以像这样计算所有订单的平均数量：

SELECT AVG(Quantity)
FROM Orders;
1
2

如果你只想计算 CustomerID 为 1 的订单的平均数量，你可以添加一个 WHERE 条件，像这样：

SELECT AVG(Quantity)
FROM Orders
WHERE CustomerID = 1;
1
2
3

注意，AVG() 函数会自动忽略 NULL 值。如果你不想忽略 NULL 值，你需要使用其他方法来处理它们，例如使用 COALESCE 函数将它们替换为 0。

# SQL COUNT() 函数

COUNT() 是 SQL 中的一个聚合函数，它用来计算一列中的行数。

COUNT() 函数的基本语法如下：

SELECT COUNT(column_name)
FROM table_name
WHERE condition;
1
2
3
  • column_name 是你想要计算行数的列的名称。
  • table_name 是你从中选择数据的表的名字。
  • WHERE condition 是一个可选的条件语句，用来限制哪些行应被计算。

例如，如果你有一个名为 Orders 的表，你可以计算表中的订单数量：

SELECT COUNT(OrderID)
FROM Orders;
1
2

如果你想计算特定客户（比如 CustomerID 为 1）的订单数量，你可以添加一个 WHERE 条件：

SELECT COUNT(OrderID)
FROM Orders
WHERE CustomerID = 1;
1
2
3

注意：

  • COUNT() 函数会忽略 NULL 值。如果你想包括 NULL 值，你应该计算表中的行数，而不是列中的值。你可以通过 COUNT(*) 来实现这一点。
  • COUNT() 函数返回的是一个 bigint 数据类型的结果。

# SQL HAVING 子句

HAVING子句在SQL中用于配合GROUP BY子句使用，其允许你对分组后的数据应用过滤条件。这与 WHERE 子句的工作方式类似，但 HAVING 主要用于处理聚合函数（如 COUNT()， AVG()， SUM()， MAX()， MIN()等）的结果。

HAVING 子句的基本语法如下：

SELECT column_name(s), aggregate_function(column_name)
FROM table_name
WHERE condition
GROUP BY column_name(s)
HAVING condition
ORDER BY column_name(s);
1
2
3
4
5
6

例如，假设你有一个名为 Orders 的表，你想找出订单数量超过 10 的客户 (CustomerID)，可以这样做：

SELECT CustomerID, COUNT(OrderID)
FROM Orders
GROUP BY CustomerID
HAVING COUNT(OrderID) > 10;
1
2
3
4

在这个例子中， GROUP BY 子句首先按 CustomerID 对订单进行分组，然后 HAVING 子句过滤出订单数量超过10的客户。

注意： HAVING 子句始终位于 GROUP BY 子句之后，但在 ORDER BY 子句之前。

# SQL CONCAT

在SQL中，CONCAT()函数用于将两个或更多的字符串组合成一个字符串。

CONCAT()函数的基本语法是：

CONCAT(string1, string2, ..., stringN)
1

例如，假设你有一个包含名(first_name)和姓(last_name)的表(person)，并且你想将这两个字段组合在一起显示全名，你可以这样做：

SELECT CONCAT(first_name, ' ', last_name) AS 'Full Name'
FROM person;
1
2

在这个例子中，' '是一个字符串，包含一个空格，用于分隔名和姓，以创建更可读的全名。

需要注意的是，CONCAT()函数的行为和支持的参数数量可能会根据不同的数据库系统（如MySQL、PostgreSQL、SQLite等）有所不同。

# SQL GROUP_CONCAT

GROUP_CONCAT() 是一个聚合函数，主要在 MySQL 数据库中使用。它会将由逗号分隔的字符串结果连接到一起。当你想要在单个字符串中创建组的值列表时，这个函数非常有用。

GROUP_CONCAT() 函数的基本语法是：

sqlCopy code
GROUP_CONCAT(expression SEPARATOR sep)
1
2

其中，expression 是你想要连接的字段，sep 是字段之间的分隔符。

例如，如果你有一个表（students），表中包含学生名字和他们所在的班级：

Student Class
Bob Math
Alice Math
Eve Science
Bob Science
Alice History

你可以使用 GROUP_CONCAT() 函数获取每个班级的学生列表：

SELECT Class, GROUP_CONCAT(Student SEPARATOR ', ')
FROM students
GROUP BY Class;
1
2
3

这将返回：

Class Students
Math Bob, Alice
Science Eve, Bob
History Alice

需要注意的是 GROUP_CONCAT() 在其他数据库系统中可能不存在，比如 PostgreSQL，或者名字可能不同，如 SQLite 中的 GROUP_CONCAT() 与 SQL Server 中的 STRING_AGG() 功能类似。

# SQL DATE_FORMAT

DATE_FORMAT函数是MySQL中特定的函数，用于将日期按照指定的格式进行格式化。它允许您将日期或日期时间值转换为具有所需格式的字符串。

DATE_FORMAT函数的语法如下：

DATE_FORMAT(date, format)
1

其中，date是要格式化的日期或日期时间值，format指定了要使用的格式模式。

例如

SELECT name, url, DATE_FORMAT(Now(),'%Y-%m-%d') AS date
FROM Websites;
1
2

# SQL IFNULL

IFNULL 是一种在 SQL 中常用的函数，用于处理 NULL 值。它接收两个参数：第一个参数是待检查的表达式或字段，第二个参数是当第一个参数为 NULL 时要返回的值。如果第一个参数不为 NULL，则返回该参数的值；如果第一个参数为 NULL，则返回第二个参数的值。

语法如下：

IFNULL(expression, value)
1
  • expression：待检查的表达式或字段。
  • value：当 expression 为 NULL 时要返回的值。

示例：

SELECT IFNULL(column_name, 'N/A') AS column_alias FROM table_name;
1

上述示例中，如果 column_name 的值为 NULL，则返回 'N/A'；否则返回 column_name 的值。

IFNULL 函数常用于处理查询结果中的 NULL 值，可以通过指定默认值来确保结果的完整性和一致性。
