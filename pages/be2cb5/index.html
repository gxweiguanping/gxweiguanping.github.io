# Linux基本操作命令

# 文件管理

# cp命令

主要用于复制文件或目录

选项 描述
-a 复制目录时保留链接、文件属性，并复制目录下的所有内容，等同于 dpR 参数组合。
-d 复制时保留链接，类似于 Windows 系统中的快捷方式。
-r, --recursive 复制目录及其所有的子目录和文件。
-i, --interactive 复制前提示确认，如果目标文件已存在，则询问是否覆盖。回答 y 时目标文件将被覆盖。
-u, --update 仅复制源文件中更新时间较新的文件。
-v, --verbose 显示详细的复制过程。
-p, --preserve 保留源文件的权限、所有者和时间戳信息。
-f, --force 强制复制，即使目标文件已存在也会覆盖，不给出提示。
-l 不复制文件，只生成链接文件。

以下是每个选项的示例用法：

参数中的 source_directory、source_file、destination_directory 和 symbolic_link 分别代表以下内容：

  • source_directory：源目录，即要复制的目录的路径。
  • source_file：源文件，即要复制的文件的路径。
  • destination_directory：目标目录，即要将文件或目录复制到的目标目录的路径。
  • symbolic_link：符号链接，用于 -d 选项中，表示要复制的链接文件的路径。

-a：复制目录时保留链接、文件属性，并复制目录下的所有内容，等同于 dpR 参数组合。

cp -a source_directory destination_directory
1

-d：复制时保留链接，类似于 Windows 系统中的快捷方式。

cp -d source_file symbolic_link
1

-r, --recursive：复制目录及其所有的子目录和文件。

cp -r source_directory destination_directory
1

-i, --interactive：复制前提示确认，如果目标文件已存在，则询问是否覆盖。

cp -i source_file destination_directory
1

-u, --update：仅复制源文件中更新时间较新的文件。

cp -u source_file destination_directory
1

-v, --verbose：显示详细的复制过程。

cp -v source_file destination_directory
1

-p, --preserve：保留源文件的权限、所有者和时间戳信息。

cp -p source_file destination_directory
1

-f, --force：强制复制，即使目标文件已存在也会覆盖，不给出提示。

cp -f source_file destination_directory
1

-l：不复制文件，只生成链接文件。

cp -l source_file symbolic_link
1

请注意，上述示例中的 source_directory、source_file、destination_directory 和 symbolic_link 需要根据您的实际情况替换为正确的路径和文件名。

# cat命令

用于连接文件并打印到标准输出设备上

选项 描述
-n, --number 由 1 开始对所有输出的行数编号。
-b, --number-nonblank 类似于 -n，但对于空白行不编号。
-s, --squeeze-blank 遇到连续两行以上的空白行，代换为一行的空白行。
-v, --show-nonprinting 使用 ^ 和 M- 符号，除了 LFD 和 TAB 之外。
-E, --show-ends 在每行结束处显示 $。
-T, --show-tabs 将 TAB 字符显示为 ^I。
-A, --show-all 等价于 -vET。
-e 等价于 "-vE" 选项。
-t 等价于 "-vT" 选项。

以下是每个选项的示例用法：

-n, --number：由 1 开始对所有输出的行数编号。 示例：将文件内容带行号输出。

cat -n file.txt
1

-b, --number-nonblank：类似于 -n，但对空白行不编号。 示例：将文件内容带行号输出，但不对空白行编号。

cat -b file.txt
1

-s, --squeeze-blank：遇到连续两行以上的空白行，代换为一行的空白行。 示例：去除文件中连续的空白行。

cat -s file.txt
1

-v, --show-nonprinting：显示非打印字符，使用 ^ 和 M- 符号，除了 LFD 和 TAB 之外。 示例：显示文件中的非打印字符。

cat -v file.txt
1

-E, --show-ends：在每行结束处显示 $ 符号。 示例：显示文件中每行的结束。

cat -E file.txt
1

-T, --show-tabs：将 TAB 字符显示为 ^I。 示例：将文件中的 TAB 字符显示为 ^I。

cat -T file.txt
1

-A, --show-all：显示非打印字符、行结束符和 TAB 字符。 示例：显示文件中的所有特殊字符。

cat -A file.txt
1

-e：等价于 "-vE" 选项。 示例：显示文件中的非打印字符和行结束符。

cat -e file.txt
1

-t：等价于 "-vT" 选项。 示例：显示文件中的非打印字符和 TAB 字符。

cat -t file.txt
1

请注意，上述示例中的 file.txt 需要替换为您实际的文件路径。

# mv命令

用来为文件或目录改名、或将文件或目录移入其它位置

选项 描述
-b 当目标文件或目录存在时，在执行覆盖前，会为其创建一个备份。
-i 如果指定移动的源目录或文件与目标的目录或文件同名，则会先询问是否覆盖旧文件。输入 y 表示直接覆盖，输入 n 表示取消该操作。
-f 如果指定移动的源目录或文件与目标的目录或文件同名，不会询问，直接覆盖旧文件。
-n 不要覆盖任何已存在的文件或目录。
-u 当源文件比目标文件新或者目标文件不存在时，才执行移动操作。

以下是每个选项的示例用法：

-b：当目标文件或目录存在时，在执行覆盖前，会为其创建一个备份。 示例：将源文件移动到目标目录，并为目标文件创建备份。

mv -b source_file destination_directory
1

-i：如果指定移动的源目录或文件与目标的目录或文件同名，则会先询问是否覆盖旧文件。 示例：将源文件移动到目标目录，如果目标文件已存在，会询问是否覆盖。

mv -i source_file destination_directory
1

-f：如果指定移动的源目录或文件与目标的目录或文件同名，不会询问，直接覆盖旧文件。 示例：将源文件移动到目标目录，如果目标文件已存在，直接覆盖。

mv -f source_file destination_directory
1

-n：不要覆盖任何已存在的文件或目录。 示例：将源文件移动到目标目录，如果目标文件已存在，则不执行移动操作。

mv -n source_file destination_directory
1

-u：当源文件比目标文件新或者目标文件不存在时，才执行移动操作。 示例：只有当源文件较新或目标文件不存在时，才执行移动操作。

mv -u source_file destination_directory
1

请注意，上述示例中的 source_file 和 destination_directory 需要替换为您实际的文件和目录路径。

# rm命令

用于删除一个文件或者目录

选项 描述
-i 删除前逐一询问确认。
-f 即使原档案属性设为唯读，也是直接删除，不需要逐一确认。
-r 递归删除目录及以下之档案

以下是每个选项的示例用法：

  1. -i：删除前逐一询问确认。 示例：删除名为 file.txt 的文件前逐一询问确认。

    rm -i file.txt
    1

  2. -f：即使原档案属性设为唯读，也是直接删除，不需要逐一确认。 示例：强制删除名为 read_only.txt 的只读文件。

    rm -f read_only.txt
    1

  3. -r：递归删除目录及以下之档案。 示例：递归删除名为 my_directory 目录及其所有内容。

    rm -r my_directory
    1

请注意，上述示例中的 file.txt、read_only.txt 和 my_directory 需要替换为您实际的文件和目录名称。当使用 -r 选项时，请谨慎操作，以免误删重要文件。

# touch命令

用于修改文件或者目录的时间属性，包括存取时间和更改时间。若文件不存在，系统会建立一个新的文件

选项 描述
-a 改变档案的读取时间记录。
-m 改变档案的修改时间记录。
-c 假如目的档案不存在，不会建立新的档案。与 --no-create 的效果一样。
-f 不使用，是为了与其他 Unix 系统的相容性而保留。
-r 使用参考档的时间记录，与 --file 的效果一样。
-d 设定时间与日期，可以使用各种不同的格式。
-t 设定档案的时间记录，格式与 date 指令相同。
--no-create 不会建立新档案。
--help 列出指令格式。
--version 列出版本讯息。

以下是每个选项的示例用法：

-a：改变档案的读取时间记录。 示例：更改名为 file.txt 的文件的读取时间记录为当前时间。

touch -a file.txt
1

-m：改变档案的修改时间记录。 示例：更改名为 file.txt 的文件的修改时间记录为当前时间。

touch -m file.txt
1

-c：假如目的档案不存在，不会建立新的档案。 示例：如果名为 new_file.txt 的文件不存在，则不会创建新文件。

touch -c new_file.txt
1

-f：不使用，为了与其他 Unix 系统的相容性而保留。

-r：使用参考档的时间记录，与 --file 的效果一样。 示例：将名为 reference_file.txt 的文件的时间记录应用于名为 target_file.txt 的文件。

touch -r reference_file.txt target_file.txt
1

-d：设定时间与日期，可以使用各种不同的格式。 示例：设定名为 file.txt 的文件的时间为特定日期和时间。

touch -d "2023-08-15 10:30:00" file.txt
1

-t：设定档案的时间记录，格式与 date 指令相同。 示例：设定名为 file.txt 的文件的时间为指定的日期和时间。

touch -t 202308151030.00 file.txt
1

--no-create：不会建立新档案。 示例：如果名为 new_file.txt 的文件不存在，则不会创建新文件，也不会修改已存在的文件。

touch --no-create new_file.txt
1

--help：列出指令格式。 示例：显示 touch 指令的帮助信息。

touch --help
1

--version：列出版本讯息。 示例：显示 touch 指令的版本信息。

touch --version
1

请注意，上述示例中的文件名和日期时间格式需要根据您的实际情况进行替换。

# find命令

用于在指定目录下查找文件和目录

选项 描述
-name pattern 按文件名查找，支持使用通配符 * 和 ?。
-type type 按文件类型查找，可以是 f（普通文件）、d（目录）、l（符号链接）等。
-size [+-]size[cwbkMG] 按文件大小查找，支持使用 + 或 - 表示大于或小于指定大小，单位可以是 c（字节）、w（字数）、b（块数）、k（KB）、M（MB）或 G（GB）。
-mtime days 按修改时间查找，支持使用 + 或 - 表示在指定天数前或后，days 是一个整数表示天数。
-user username 按文件所有者查找。
-group groupname 按文件所属组查找。

以下是每个选项的示例用法：

  1. -name pattern：按文件名查找，支持使用通配符 * 和 ?。 示例：查找当前目录下所有以 .txt 结尾的文件。

    find . -name "*.txt"
    1

  2. -type type：按文件类型查找，可以是 f（普通文件）、d（目录）、l（符号链接）等。 示例：查找当前目录下所有的目录。

    find . -type d
    1

  3. -size [+-]size[cwbkMG]：按文件大小查找，支持使用 + 或 - 表示大于或小于指定大小，单位可以是 c（字节）、w（字数）、b（块数）、k（KB）、M（MB）或 G（GB）。 示例：查找当前目录下所有大小大于 1MB 的文件。

    find . -size +1M
    1

  4. -mtime days：按修改时间查找，支持使用 + 或 - 表示在指定天数前或后，days 是一个整数表示天数。 示例：查找当前目录下所有在最近 7 天内修改过的文件。

    find . -mtime -7
    1

  5. -user username：按文件所有者查找。 示例：查找当前目录下所有属于用户 john 的文件。

    find . -user john
    1

  6. -group groupname：按文件所属组查找。 示例：查找当前目录下所有属于组 developers 的文件。

    find . -group developers
    1

请注意，上述示例中的选项和参数需要根据您的实际情况进行替换。

# 文档编辑

# vi/vim命令

vim 是从 vi 发展出来的一个文本编辑器

vi/vim 共分为三种模式，命令模式（Command Mode）、输入模式（Insert Mode）和命令行模式（Command-Line Mode）。

# 命令模式

这是 Vi/Vim 默认进入的模式,在这个模式下，可以执行诸如移动光标、复制、粘贴、删除、搜索等操作。

命令模式下按下不同的按键会触发不同的命令，这些命令通常会影响文本编辑器的状态或者执行特定的操作。

在命令模式中，可以使用以下常用按键：

  • h, j, k, l：光标移动命令，分别向左、下、上、右移动。
  • x：删除光标所在位置的字符。
  • yy：复制一行文本。
  • dd：删除一行文本。
  • /：开始搜索文本的命令。
    • 当你按下 / 后，Vi/Vim 会进入搜索模式，此时你可以输入你想要查找的文本内容，并按下回车键。编辑器会在当前光标位置之后开始搜索匹配该文本的内容。
    • 一旦按下回车键确认搜索内容，Vi/Vim 会立即跳转到第一个匹配项的位置，并将光标定位到该处。
    • 如果想要找到下一个匹配项，可以按下 n（下一个）键，在文档中依次向后查找匹配项。相反，按下 N（前一个）键可以向前查找匹配项。
  • :w ： 保存文件。
  • u：撤销上一次操作。
  • :q ：退出 Vim 编辑器。
  • :q! ：强制退出Vim 编辑器，不保存修改。

# 输入模式

在命令模式下按下 i 进入插入模式的命令后，就会进入插入模式。在这个模式下，可以直接输入文本内容，并且按下 Esc 键可以返回到命令模式。

在输入模式中，可以使用以下常用按键：

  • 字符按键及Shift组合：用于输入文本内容。
  • ENTER：回车键，用于换行。
  • BACK SPACE：退格键，删除光标前一个字符。
  • DEL：删除键，删除光标后一个字符。
  • 方向键：在文本中移动光标，可调整光标位置。
  • HOME/END：将光标移动到行首/行尾。
  • Page Up/Page Down：上/下翻页，便于查看长文档。
  • Insert：切换光标为输入/替换模式。光标形状改变，竖线表示输入模式，下划线表示替换模式。
  • ESC：退出输入模式，切换至命令模式。这是一种方式，允许你在完成输入后返回普通模式，执行其他操作如移动光标、复制文本等。

# 命令行模式

在底线命令模式中，基本的命令有（已经省略了冒号）：

  • :w：保存文件。
  • :q：退出 Vim 编辑器。
  • :wq：保存文件并退出 Vim 编辑器。
  • :q!：强制退出Vim编辑器，不保存修改。

# 设置文本格式为 Unix 格式

可以通过执行以下命令完成：

:set ff=unix
1

这个命令的含义是设置文件的格式（file format）为 Unix。在 Vim 中，ff 代表文件格式，unix 表示使用 LF（换行符）作为行结束符。

# 查看当前文件的格式

可以在 Vim 的普通模式下执行以下命令：

:set ff?
1

这个命令会显示当前文件的格式信息。在命令模式下输入 :set ff? 并按下回车键后，Vim 会显示文件的格式，可能是 unix、dos 或 mac，分别代表 Unix、Windows 和 Mac 格式。

# grep命令

用于查找文件里符合条件的字符串或正则表达式

选项 描述
-i 忽略大小写进行匹配。
-v 反向查找，只打印不匹配的行。
-n 显示匹配行的行号。
-r 递归查找子目录中的文件。
-l 只打印匹配的文件名。
-c 只打印匹配的行数。

以下是每个选项的示例用法：

  1. -i：忽略大小写进行匹配。 示例：在文件 file.txt 中查找所有包含字符串 apple 的行，忽略大小写。

    grep -i "apple" file.txt
    1

  2. -v：反向查找，只打印不匹配的行。 示例：在文件 file.txt 中查找所有不包含字符串 banana 的行。

    grep -v "banana" file.txt
    1

  3. -n：显示匹配行的行号。 示例：在文件 file.txt 中查找包含字符串 orange 的行，并显示行号。

    grep -n "orange" file.txt
    1

  4. -r：递归查找子目录中的文件。 示例：在目录 my_directory 及其子目录中递归查找包含字符串 pear 的文件。

    grep -r "pear" my_directory
    1

  5. -l：只打印匹配的文件名。 示例：在目录 my_directory 中查找包含字符串 grape 的文件，并只打印文件名。

    grep -l "grape" my_directory/*
    1

  6. -c：只打印匹配的行数。 示例：在文件 file.txt 中查找包含字符串 cherry 的行，并打印匹配的行数。

    grep -c "cherry" file.txt
    1

请注意，上述示例中的字符串和文件名需要根据您的实际情况进行替换。

# 磁盘管理

# ls命令

Linux ls（英文全拼： list directory contents）命令用于显示指定工作目录下之内容（列出目前工作目录所含的文件及子目录)。

选项 描述
-a 显示所有文件及目录 (包括以“.”开头的隐藏文件)
-l 使用长格式列出文件及目录的详细信息
-r 将文件以相反次序显示 (默认依英文字母次序)
-t 根据最后的修改时间排序

# free命令

显示内存状态

语法格式

free [-bkmotV][-s <间隔秒数>]
1

常用选项

参数 描述
-b 以字节为单位显示内存使用情况。
-k 以KB为单位显示内存使用情况。
-m 以MB为单位显示内存使用情况。
-h 以人类可读的格式显示内存大小，使用合适的单位（如 KB、MB、GB 等）。
-o 不显示缓冲区调节列。
-s<间隔秒数> 持续观察内存使用状况，指定刷新时间间隔，以秒为单位，表示每隔多少秒刷新一次内存使用情况。
-t 显示内存总和列。
-V 显示版本信息。

默认情况下，free 命令会输出如下信息：

  • total 列示系统总的物理内存和交换空间的大小。
  • used 列示已使用的物理内存和交换空间的大小。
  • free 列示空闲的物理内存和交换空间的大小。
  • shared 列示被共享的物理内存的大小。
  • buff/cache 列示被缓冲和缓存的内存大小。
  • available 仅在最新的内核版本中出现，它表示系统可以立即分配给进程的近似空闲内存大小。

Mem（内存）：

  • Mem 是指物理内存（RAM），是计算机中用于临时存储数据和程序的地方。在操作系统中，运行的程序和系统需要的数据都会存储在物理内存中。物理内存的大小直接影响着系统能够同时运行的程序数量和性能。

Swap（交换空间）：

  • Swap 是指硬盘上的一块用于扩展虚拟内存的空间。当物理内存不足以满足系统当前运行程序的需求时，部分暂时不使用或使用频率低的内存内容会被存储到交换空间中。这样可以释放物理内存，为更重要或更活跃的程序提供更多空间。交换空间的使用虽然可以帮助系统继续运行，但相对于物理内存来说，硬盘的读写速度要慢得多，因此使用交换空间可能会导致系统性能下降。

free -m
1

free -h
1

列出压缩文件的内容列表：

unzip -l example.zip
1

安静模式解压缩：

unzip -q example.zip
1

不询问直接覆盖已存在的文件：

unzip -o example.zip
1

请注意，unzip 命令只适用于解压缩 ZIP 格式的压缩文件，如果需要解压其他格式的压缩文件，可能需要使用其他相关的命令。

# ssh命令

用于在网络上远程登录或执行命令的安全协议和工具

语法如下

ssh [选项] [用户名@]主机名 [命令]
1

常用选项

选项 说明
-p port 指定远程主机的端口号，默认为 22。
-l login 指定要使用的登录用户名。
-i file 指定用于身份验证的私钥文件路径。
-N 仅建立连接但不执行远程命令。
-f 在后台运行 ssh 连接。
-L [bind_address:]port:host:hostport 将本地端口转发到远程主机。
-R [bind_address:]port:host:hostport 将远程主机端口转发到本地主机。
-D port 创建动态端口转发，允许通过 ssh 连接的代理服务器进行 SOCKS 代理。
-C 启用压缩，加速数据传输。

远程登录到主机 example.com：

ssh username@example.com
1

使用指定的私钥文件进行远程登录：

ssh -i ~/.ssh/private_key.pem username@example.com
1

在后台运行远程连接：

ssh -f username@example.com
1

将本地端口转发到远程主机的指定端口：

ssh -L 8080:localhost:80 username@example.com
1

创建动态端口转发，用于代理服务器：

ssh -D 1080 username@example.com
1

通过 ssh 命令，可以安全地连接到远程主机并执行命令，进行文件传输等操作。

# yum命令

基于 RPM 包管理系统的 Linux 操作系统上进行软件包安装、更新和管理的命令行工具-h 显示帮助信息

语法

yum [选项] [命令] [软件包名]
1

常用选项

选项 说明
install 安装一个或多个软件包。
update 更新已安装的软件包。
remove 卸载一个或多个已安装的软件包。
search 在软件仓库中搜索软件包。
list 列出已安装或可用的软件包。
info 显示软件包的详细信息。
provides 查找提供指定文件的软件包。
groupinfo 显示软件包组的详细信息。
grouplist 列出可用的软件包组。
clean 清理缓存和临时文件。
repolist 列出可用的仓库列表。

例子

安装软件包：

yum install package-name
1

更新已安装的软件包：

yum update
1

卸载软件包：

yum remove package-name
1

搜索软件包：

yum search keyword
1

列出已安装的软件包：

yum list installed
1

⚠️小贴士

有的时候由于网络安全等原因，部分服务器不被允许上公网。这就导致了，我们用yum安装一些包的时候，是无法执行成功的。我们可以通过创建一个本地的yum仓库，来解决这一问题。

我们可以使用yum-downloadonly 插件来下载软件包，然后将这些软件包放在本地，以便在没有公网连接的服务器上进行安装。

1、安装 yum-downloadonly 插件：

首先，您需要安装 yum-downloadonly 插件。通过以下命令在系统上安装它：

sudo yum install yum-downloadonly
1

2、使用 yum-downloadonly 下载软件包：

使用以下命令来下载特定软件包到指定目录。例如，下载 package-name 软件包到 /path/to/download 目录：

sudo yum install --downloadonly --downloaddir=/path/to/download package-name
1

这将会下载软件包及其所有依赖关系到指定的目录。

3、在本地服务器上使用下载的软件包：

将下载的软件包复制到目标服务器，然后通过本地路径或本地yum仓库的方式进行安装。例如：

  • 通过本地路径安装：

    sudo yum install /path/to/download/package-name.rpm
    1

  • 通过本地yum仓库安装（需要先配置本地yum仓库）：

    sudo yum install package-name
    1

tail --pid=12345 -f logfile
